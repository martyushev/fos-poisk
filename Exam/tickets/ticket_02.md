## Часть 1. Теоретический вопрос (30 баллов)

**Что такое инвертированный индекс? Как он строится и используется для ускорения поиска?**

*Рекомендация по ответу:*

- **Определение:** структура данных: термин → список документов (постинг-лист)
- **Построение:** токенизация → нормализация → сортировка → слияние
- **Постинг-лист:** doc_id, позиции, частота
- **Ускорение:** пересечение постинг-листов вместо сканирования всех документов
- **Оптимизация:** сжатие, skip-листы


## Часть 2. Практическая задача (40 баллов)

**Расчёт Precision, Recall, F-меры**

Релевантность (10 документов): [1, 0, 0, 1, 0, 1, 0, 0, 1, 0]

**a) Precision@5 и Recall@5:**
- Топ-5: [1, 0, 0, 1, 0] → 2 релевантных
- Всего релевантных: 4
- Precision@5 = 2/5 = 0.4
- Recall@5 = 2/4 = 0.5

**б) F1-мера при k=5:**
- F1 = 2 × (0.4 × 0.5) / (0.4 + 0.5) = 0.4 / 0.9 ≈ 0.444


## Часть 3. Дополнительный вопрос (30 баллов)

**Объясните принцип работы TF-IDF.**

- **TF:** частота термина в документе
- **IDF:** log(N/df) — обратная частота документа
- **Смысл:** редкие термины важнее частых
- **Ограничения:** не учитывает длину документа, семантику
