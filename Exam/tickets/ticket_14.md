# Экзаменационный билет № 14

## Часть 1. Теоретический вопрос (30 баллов)

**Что такое LSA/LSI? Как работает сингулярное разложение (SVD) для снижения размерности?**

*Рекомендация по ответу:*

- **LSA/LSI:** Latent Semantic Analysis / Indexing — выявление скрытых связей между терминами
- **Матрица:** термин-документ (TF или TF-IDF)
- **SVD:** разложение матрицы A = U × Σ × V^T
- **Снижение размерности:** усечение SVD до k компонент
- **Эффект:** выявление семантических связей, сглаживание шума
- **Ограничения:** размывает точные идентификаторы, не решает полисемию полностью


## Часть 2. Практическая задача (40 баллов)

**Расчёт силуэта (Silhouette Score)**

Объясните, как вычисляется коэффициент силуэта и как он помогает выбрать k.

**Формула:**
- s(i) = (b(i) - a(i)) / max(a(i), b(i))
- a(i): среднее расстояние до точек своего кластера
- b(i): среднее расстояние до точек соседнего кластера

**Интерпретация:**
- s(i) ∈ [-1, 1]
- Близко к 1: хорошо
- Близко к 0: на границе
- Близко к -1: плохо


## Часть 3. Дополнительный вопрос (30 баллов)

**Почему LSA может быть опасен для поиска по юридическим документам?**

- **Проблема:** размывает точные идентификаторы (номера статей)
- **Пример:** «статья 106 УК РФ» → LSA может найти скрытый смысл вместо точного совпадения
- **Решение:** гибридный подход (точный поиск + семантический)
