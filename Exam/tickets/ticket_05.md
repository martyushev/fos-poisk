# Экзаменационный билет № 5

## Часть 1. Теоретический вопрос (30 баллов)

**В чём отличие Bi-Encoder от Cross-Encoder? В каких сценариях какой подход предпочтительнее?**

*Рекомендация по ответу:*

- **Bi-Encoder:** раздельное кодирование [query] и [doc], эмбеддинги пре-вычисляются; быстрый поиск через ANN
- **Cross-Encoder:** совместное кодирование [query+doc], учитывает взаимодействия; медленный (O(N) прогонов)
- **Trade-off:** Cross-Encoder — высокое качество, высокая latency; Bi-Encoder — низкая latency, среднее качество
- **Сценарии:** Bi-Encoder для candidate generation, Cross-Encoder для reranking
- **Гибрид:** двухэтапная схема (Bi-Encoder → Cross-Encoder)


## Часть 2. Практическая задача (40 баллов)

**Проектирование LTR-пайплайна**

Даны логи поиска (query, document, clicks, position).

**Этапы пайплайна:**
1. **Генерация признаков:** текстовые (BM25), поведенческие (CTR × 1/log(1+position)), статические (длина документа)
2. **Группировка:** по query_id
3. **Baseline:** pointwise (регрессия)
4. **Оптимизация:** listwise (LGBMRanker с objective='lambdarank')
5. **Оценка:** NDCG@5, сравнение с baseline


## Часть 3. Дополнительный вопрос (30 баллов)

**Что такое Position Bias и как его корректировать?**

- **Position Bias:** пользователи чаще кликают на документы в топе, даже если они менее релевантны
- **Коррекция:** позиционные признаки в модели, IPW (Inverse Propensity Weighting), сглаживание CTR × 1/log(1+position)
