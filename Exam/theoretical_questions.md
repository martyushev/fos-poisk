# Полный перечень теоретических вопросов к экзамену

## Раздел 1. Архитектура поисковых систем (вопросы 1–2)

**1. Опишите классическую архитектуру поисковой системы. Какие компоненты входят в её состав и как они взаимодействуют?** [Модуль 1, занятие 1.1]

*Рекомендация по ответу:*
- **Краулер:** функции обхода веб-страниц, politeness policy, deduplication
- **Индексатор:** построение инвертированного индекса, токенизация, нормализация
- **Ранкер:** этапы ранжирования, классические и ML-модели
- **Сервер выдачи:** обработка запросов, сниппеты, latency
- **Схема взаимодействия компонентов:** краулер → индексатор → индекс → ранкер → сервер выдачи

**2. Что такое инвертированный индекс? Как он строится и как используется для ускорения поиска?** [Модуль 1, занятие 1.1]

*Рекомендация по ответу:*
- **Определение:** структура данных, сопоставляющая термин со списком документов
- **Структура:** термин → постинг-лист (doc_id, позиции, частота)
- **Построение:** токенизация → нормализация → сортировка → слияние
- **Использование:** пересечение постинг-листов при обработке запросов
- **Оптимизация:** сжатие, skip-листы, разбиение по сегментам

## Раздел 2. Классические модели ранжирования (вопросы 3–4)

**3. Объясните принцип работы TF-IDF. Как вычисляются TF и IDF? Какие проблемы решает эта модель и какие у неё ограничения?** [Модуль 1, занятие 1.1]

*Рекомендация по ответу:*
- **TF:** частота термина в документе (с нормализацией или без)
- **IDF:** обратная частота документа — log(N/df)
- **Формула:** TF-IDF = TF × IDF
- **Смысл:** редкие термины важнее частых
- **Ограничения:** не учитывает длину документа, насыщение частоты, семантику

**4. Сравните TF-IDF и BM25. В чём отличие формулы BM25, какие параметры в ней используются и как они влияют на ранжирование?** [Модуль 1, занятие 1.1]

*Рекомендация по ответу:*
- **Формула BM25:** сумма по терминам IDF × (tf × (k1+1)) / (tf + k1 × (1 — b + b × (dl/avgdl)))
- **Параметр k1:** насыщение частоты термина (типичное значение 1.2–2.0)
- **Параметр b:** нормализация длины документа (типичное значение 0.75)
- **Преимущества:** учёт длины документа, насыщение частоты

## Раздел 3. Learning to Rank (вопросы 5–6)

**5. Объясните различия между pointwise, pairwise и listwise подходами к обучению ранжированию. Приведите примеры функций потерь для каждого подхода.** [Модуль 2, занятие 2.1]

*Рекомендация по ответу:*
- **Pointwise:** регрессия/классификация для каждого документа; MSE, LogLoss
- **Pairwise:** попарное сравнение документов; RankNet (кросс-энтропия), RankSVM
- **Listwise:** оптимизация метрики списка; LambdaRank, ListNet, ListMLE
- **Сравнение:** качество (listwise ≥ pairwise ≥ pointwise), сложность (обратная зависимость)

**6. В чём отличие Bi-Encoder от Cross-Encoder? В каких сценариях какой подход предпочтительнее? Опишите trade-off между качеством и latency.** [Модуль 2, занятие 2.2]

*Рекомендация по ответу:*
- **Bi-Encoder:** раздельное кодирование [query] и [doc], эмбеддинги пре-вычисляются; быстрый поиск через ANN
- **Cross-Encoder:** совместное кодирование [query+doc], учитывает взаимодействия; медленный (O(N) прогонов)
- **Trade-off:** Cross-Encoder — высокое качество, высокая latency; Bi-Encoder — низкая latency, среднее качество
- **Сценарии:** Bi-Encoder для candidate generation, Cross-Encoder для reranking
- **Гибридная архитектура:** двухэтапная схема (Retriever → Re-ranker)

## Раздел 4. Нейросетевые подходы (вопросы 7–8)

**7. Объясните механизм Self-Attention в архитектуре Transformer. Почему он превосходит рекуррентные архитектуры (RNN/LSTM) для задач поиска?** [Модуль 4, занятие 4.1]

*Рекомендация по ответу:*
- **Query, Key, Value:** вычисление весов внимания
- **Формула:** Attention(Q,K,V) = softmax(Q×K^T/√d_k) × V
- **Multi-head attention:** несколько параллельных голов внимания
- **Преимущества:** параллелизм (без последовательной обработки), длинные зависимости, контекст
- **Positional Encoding:** добавление информации о порядке слов

**8. Что такое BERT? Опишите архитектуру, этапы предобучения (MLM, NSP) и применение в информационном поиске.** [Модуль 4, занятие 4.1]

*Рекомендация по ответу:*
- **Архитектура:** Transformer Encoder (12/24 слоя)
- **MLM (Masked Language Model):** маскирование 15% токенов, предсказание по контексту
- **NSP (Next Sentence Prediction):** предсказание следующего предложения
- **CLS-токен:** агрегированное представление для классификации
- **Применение в поиске:** эмбеддинги документов, ранжирование, понимание запроса

## Раздел 5. Векторный поиск и метрики (вопросы 9–10)

**9. Объясните принципы векторного поиска. Какие алгоритмы ANN (HNSW, IVF) используются и как они работают?** [Модуль 4, занятие 4.3]

*Рекомендация по ответу:*
- **Эмбеддинги:** преобразование текста в числовые векторы
- **Проблема:** полный перебор O(N·d) для миллионов документов
- **ANN (Approximate Nearest Neighbors):** приближённый поиск с контролируемой точностью
- **HNSW (Hierarchical Navigable Small World):** многослойная графовая структура
- **IVF (Inverted File Index):** кластеризация + пробирование
- **Библиотеки:** FAISS, ScaNN

**10. Какие метрики используются для оценки качества ранжирования? Объясните формулы Precision, Recall, MAP, MRR, NDCG и их интерпретацию.** [Модуль 1, занятие 1.1; Модуль 4, занятие 4.3]

*Рекомендация по ответу:*
- **Precision:** доля релевантных среди выданных
- **Recall:** доля найденных среди всех релевантных
- **MAP (Mean Average Precision):** средняя точность с учётом позиции
- **MRR (Mean Reciprocal Rank):** обратный ранг первого релевантного
- **NDCG (Normalized Discounted Cumulative Gain):** взвешенная по позиции релевантность, нормализованная по идеальному порядку

## Раздел 6. Инженерия признаков и понимание запроса (вопросы 11–12)

**11. Какие типы признаков используются в задачах ранжирования? Приведите примеры текстовых, поведенческих, статических и составных признаков.** [Модуль 2, занятие 2.1]

*Рекомендация по ответу:*
- **Текстовые:** BM25, TF-IDF, косинусное сходство, эмбеддинги
- **Поведенческие:** CTR, dwell time, история кликов, время просмотра
- **Статические:** длина документа, возраст, язык, PageRank
- **Составные (cross-features):** произведения/суммы/комбинации признаков
- **Трансформации:** логарифмирование, bucketization, нормализация

**12. Опишите пайплайн понимания запроса (Query Understanding). Какие задачи в него входят: spell correction, NER, intent classification, query expansion?** [Модуль 4, занятие 4.2]

*Рекомендация по ответу:*
- **Spell correction:** символьные n-граммы, нейросетевой корректор (T5-based)
- **NER (Named Entity Recognition):** извлечение сущностей (BIO-разметка, BERT)
- **Intent classification:** классификация намерений (fine-tuned BERT, 5–7 классов)
- **Query expansion:** генерация синонимов (doc2query, BERT masked LM)
- **Интеграция:** последовательная или параллельная обработка

## Раздел 7. Обработка естественного языка (вопросы 13–15)

**13. Какие этапы предобработки текста применяются в поисковых системах? Объясните токенизацию, лемматизацию, стемминг и удаление стоп-слов.** [Модуль 3, занятие 3.1]

*Рекомендация по ответу:*
- **Токенизация:** разбиение текста на токены (слова, пунктуация)
- **Нормализация:** приведение к нижнему регистру, удаление HTML-тегов
- **Стоп-слова:** удаление общеупотребительных слов (кроме медицинских/юридических доменов)
- **Стемминг:** обрезание окончаний (Porter, Snowball)
- **Лемматизация:** приведение к начальной форме (pymorphy2, WordNet)
- **Фиксация изменения размера словаря**

**14. Что такое LSA/LSI? Как работает сингулярное разложение (SVD) для снижения размерности и выявления скрытых связей между терминами?** [Модуль 3, занятие 3.1]

*Рекомендация по ответу:*
- **Матрица термин-документ:** TF или TF-IDF
- **SVD:** разложение матрицы на U × Σ × V^T
- **Снижение размерности:** усечение SVD до k компонент
- **Выявление скрытых связей:** совместная встречаемость терминов
- **Ограничения:** размывает точные идентификаторы, не решает полисемию полностью

**15. Объясните методы кластеризации текстов (KMeans). Как выбирается оптимальное число кластеров? Для чего используется кластеризация в поисковых системах?** [Модуль 3, занятие 3.2]

*Рекомендация по ответу:*
- **KMeans:** итерационный алгоритм с центрами кластеров
- **Выбор k:** метод локтя (inertia), коэффициент силуэта (silhouette score)
- **Интерпретация:** топ-10 слов по центру кластера
- **Применение:** группировка документов по темам, категоризация, анализ тем
- **Визуализация:** t-SNE/UMAP для 2D-проекции

## Раздел 8. Этические и специализированные аспекты поиска (вопросы 16–20)

**16. Какие этические проблемы существуют в информационном поиске? Объясните понятия bias, информационная справедливость, конфиденциальность данных.** [Модуль 5, занятие 5.2]

*Рекомендация по ответу:*
- **Bias:** систематическое смещение результатов (пол, раса, география)
- **Информационная справедливость:** равный доступ к информации для всех
- **Конфиденциальность:** защита данных пользователей (анонимизация, GDPR)
- **Прозрачность:** объяснимость ранжирования (SHAP, LIME)
- **Информационный пузырь:** ограничение разнообразия информации

**17. Что такое персонализация поиска? Опишите методы построения профилей пользователей и подходы к персонализированному ранжированию.** [Модуль 5, занятие 5.1]

*Рекомендация по ответу:*
- **Определение:** адаптация результатов под пользователя
- **Явные сигналы:** жанры, рейтинги, избранное
- **Неявные сигналы:** история просмотров, клики, dwell time
- **Профиль пользователя:** эмбеддинг предпочтений
- **Персонализированное ранжирование:** комбинация глобального и персонализированного скора
- **Оценка:** персонализированный NDCG, diversity, novelty

**18. Что такое мультимодальный поиск? Опишите подходы к поиску по изображениям (CBIR, deep visual embeddings) и голосовому поиску (ASR).** [Модуль 5, занятие 5.2]

*Рекомендация по ответу:*
- **Мультимодальный поиск:** поиск по данным разных модальностей (текст, изображения, аудио, видео)
- **CBIR (Content-Based Image Retrieval):** поиск по визуальному содержимому
- **Deep visual embeddings:** CNN для извлечения признаков изображений
- **Голосовой поиск:** ASR (Speech-to-Text) + NLP
- **Мультимодальные модели:** CLIP, ViT, мультиязычные эмбеддинги

**19. В чём особенности корпоративного поиска? Какие источники данных и форматы документов характерны для него?** [Модуль 5, занятие 5.2]

*Рекомендация по ответу:*
- **Особенности:** разнородные источники, права доступа, безопасность
- **Источники:** Confluence, Sharepoint, электронные письма, чаты, файловые системы
- **Форматы:** HTML, PDF, Word, Excel, JSON, сообщения чатов
- **Проблемы:** извлечение чистого текста из вложений, иерархия структуры
- **Сложности:** учёт прав доступа в ранжировании

**20. Что такое Position Bias? Как он влияет на обучение ранжированию и какие методы используются для его коррекции?** [Модуль 2, занятие 2.1]

*Рекомендация по ответу:*
- **Position Bias:** пользователи склонны кликать на документы в топе, даже если они менее релевантны
- **Влияние:** переоценка релевантности позиционных документов
- **Коррекция:** использование позиционных признаков в модели
- **IPW (Inverse Propensity Weighting):** взвешивание примеров по обратной вероятности клика
- **Сглаживание:** CTR × 1/log(1 + position)
