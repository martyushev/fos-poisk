# Полный перечень практических задач к экзамену

## Задача 1. Расчёт метрик ранжирования

По заданному ранжированному списку документов и экспертной оценке релевантности рассчитайте:

- Precision@k и Recall@k
- NDCG@k
- MRR@k
- MAP

**Пример данных:** Ранжированный список из 5–10 документов с бинарной или градуированной релевантностью.

**Ожидаемый результат:** Демонстрация промежуточных вычислений (DCG, IDCG, идеальный порядок) и финальных значений метрик.

**Связанные темы:** Метрики качества поиска (Модуль 1), Оценка ранжирования (Модуль 4).

## Задача 2. Проектирование архитектуры поисковой системы

Для заданного домена (интернет-магазин, новостной агрегатор, медицинский поиск, юридический поиск) предложите архитектуру поисковой системы.

**Ожидаемый результат:**
- Выбор модели для первого этапа (candidate generation)
- Выбор модели для второго этапа (reranking)
- Схема индексации
- Пайплайн обработки запроса
- Обоснование выбора подходов

**Связанные темы:** Архитектура поисковых систем (Модуль 1), Bi-Encoder vs Cross-Encoder (Модуль 2).

## Задача 3. Генерация признаков для ранжирования

Для заданного набора данных (логи поиска) предложите систему признаков для обучения модели ранжирования.

**Ожидаемый результат:**
- Минимум 3 текстовых признака
- Минимум 2 поведенческих признака с учётом Position Bias
- Минимум 1 статический признак
- Минимум 1 составной признак (cross-feature)
- Обоснование значимости каждого признака

**Связанные темы:** Инженерия признаков (Модуль 2).

## Задача 4. Анализ ошибок ранжировщика

По заданным примерам ошибок нейросетевого ранжировщика проведите анализ и предложите способы их устранения.

**Ожидаемый результат:**
- Классификация ошибок (лексические, семантические, поведенческие)
- Предложение методов устранения для каждого типа ошибок
- Обоснование приоритетности исправлений

**Связанные темы:** Обучение ранжированию (Модуль 2), Нейросетевые подходы (Модуль 4).

## Задача 5. Проектирование пайплайна Query Understanding

Для заданного типа поисковой системы спроектируйте пайплайн обработки запроса.

**Ожидаемый результат:**
- Выбор методов spell correction (символьные n-граммы, нейросетевой корректор)
- Выбор модели для NER
- Выбор модели для intent classification
- Выбор модели для query expansion
- Схема интеграции модулей
- Оценка влияния каждого модуля на качество

**Связанные темы:** Понимание запроса (Модуль 4).

## Задача 6. Этический аудит поисковой системы

Для заданной поисковой системы проведите этический аудит.

**Ожидаемый результат:**
- Выявление 3 этических проблем
- Описание пострадавших и уязвимых сценариев
- Предложение метрик для измерения проблем
- Предложение методов смягчения

**Связанные темы:** Этические проблемы поиска (Модуль 5).

## Задача 7. Проектирование персонализированного поиска

Для заданного домена (онлайн-кинотеатр, интернет-магазин, новостной агрегатор) спроектируйте персонализированный поиск.

**Ожидаемый результат:**
- Типы собираемых данных (явные и неявные сигналы)
- Структура профиля пользователя
- Архитектура персонализированного ранжирования
- Методы оценки качества персонализации
- Сценарий A/B-теста

**Связанные темы:** Персонализация поиска (Модуль 5).

## Задача 8. Расчёт метрик с учётом Position Bias

По заданным данным о позициях документов и кликах скорректируйте метрики с учётом Position Bias.

**Ожидаемый результат:**
- Расчёт сырых метрик (CTR по позициям)
- Применение коррекции (IPW или другие методы)
- Сравнение скорректированных и нескорректированных метрик

**Связанные темы:** Position Bias (Модуль 2).
