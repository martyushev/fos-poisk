# Экзаменационные билеты по дисциплине «Поисковые системы с ИИ»  
   
### Билет №1
1.	Общая структура поисковой системы. Компоненты и их назначение.
2.	Векторное представление документов.
3.	Дан корпус из 5 документов и 3 запроса. Постройте инвертированный индекс для коллекции. Для одного из запросов вычислите TF-IDF веса терминов и ранжируйте документы по косинусному сходству с запросом.

### Билет №2
1.	Модель инвертированного индекса. Эффективность и ограничения.
2.	Составляющие комплексной оценки релевантности.
3.	Дан список из 6 документов с размеченными экспертными оценками релевантности (0 – 2). Вычислите метрики NDCG@5 и MAP для двух гипотетических выдач (ранжирование по BM25 и случайное перемешивание). Сравните результаты.

### Билет №3
1.	Преимущества и недостатки классических моделей ранжирования.
2.	Применение эмбеддингов в поиске.
3.	Для заданной коллекции документов постройте векторные представления для 10 ключевых терминов. Найдите 5 ближайших соседей для заданного слова по косинусной близости и объясните результат.

### Билет №4
1.	Различия между статическими и динамическими признаками ранжирования.
2.	Выбор подходящего набора признаков в задаче ранжирования.
3.	Предложите 3 новых динамических признака для ранжирования, которые можно извлечь из логов кликов пользователей. Опишите, как каждый признак вычисляется, и обоснуйте, почему он может улучшить качество ранжирования.

### Билет №5
1.	Ранжирование как задача машинного обучения. Отличия подходов pointwise, pairwise, listwise.
2.	Применение нейросетевых методов для снижения доли нерелевантных результатов.
3.	Дано 5 запросов с помеченной релевантностью (0 – 1). Обучите логистическую регрессию вручную (одна итерация градиентного спуска) с двумя признаками: TF-IDF и длина документа. Покажите расчёт градиентов и обновление весов.

### Билет №6
1.	Алгоритм PageRank: сущность, преимущества и слабые стороны.
2.	Роль обратных ссылок в определении популярности ресурса.
3.	Для заданного веб-графа из 5 страниц выполните одну итерацию алгоритма PageRank с damping factor = 0.85. Рассчитайте веса страниц и определите самую авторитетную.

### Билет №7
1.	Архитектурные особенности интеллектуальных поисковых систем.
2.	Организация поиска по изображениям и специфика обработки изображений в индексе.
3.	Дан запрос «поиск похожих изображений по картинке». Опишите архитектуру системы (компоненты, модели, индексы) для организации поиска по изображениям, указав, как извлекаются признаки и как они сравниваются.

### Билет №8
1.	Решение проблемы отсутствия нужных результатов в выдаче.
2.	Семантический поиск и разработка схем для определения истинного значения запроса.
3.	Реализуйте в псевдокоде функцию, которая принимает запрос и расширяет его синонимами (синонимы заданы словарём). Покажите на примере запроса «купить ноутбук» финальный расширенный запрос и объясните, как это помогает решить проблему отсутствия результатов.

### Билет №9
1.	Механизм работы веб-краулеров и проблема дубликатов документов.
2.	Мультилингвистический поиск и стратегии его реализации.
3.	Напишите псевдокод для краулера, который обходит сайт, избегая дубликатов страниц. Покажите, как он обрабатывает ссылки и как определяет, что страница уже была проиндексирована.

### Билет №10
1.	Метрики оценки качества поиска.
2.	Понятия документоориентированного и объектно-ориентированного поиска.
3.	Предложите 4 критерия для комплексной оценки релевантности документа запросу, включая семантические и поведенческие факторы. Для каждого критерия опишите, как его измерить и как объединить в итоговый скор.

### Билет №11
1.	Подходы к извлечению значимых признаков для машинного обучения в ранжировании.
2.	Современные техники кластеризации документов для ускорения поиска.
3.	Для коллекции из 50 текстов выполните кластеризацию на 3 кластера. Укажите центроиды в виде списка ключевых слов и объясните, как это можно использовать для ускорения поиска.

### Билет №12
1.	Понятие нейросетевого ранжирования. Примеры архитектур.
2.	Использование нейросетевых подходов для анализа эмоций в текстах.
3.	Предложите архитектуру нейросетевого ранкера для задачи ранжирования. Опишите входные данные, слои, функцию потерь и способ интеграции с существующей поисковой системой.

### Билет №13
1.	Понятия корневого домена и субдоменов в контексте поиска.
2.	Отличительные черты геолокационного поиска и задачи локализации выдач.
3.	Дан запрос с географической привязкой. Опишите, как система определяет местоположение пользователя, как учитывается гео-признак в ранжировании и как агрегируются результаты для выдачи.

### Билет №14
1.	Особенности применения глубокого обучения в поисковых системах.
2.	Выделение паттернов и скрытых связей.
3.	Постройте схему латентно-семантического индексирования для трёх документов с заданной матрицей терм-документ. Выполните SVD-разложение и найдите скрытые темы.

### Билет №15
1.	Основные задачи обработки естественного языка в контексте поиска.
2.	Стратегии агрегации и отображения коммерческих результатов поиска.
3.	Поисковый индекс содержит 10⁶ документов. Оцените время доступа для инвертированного индекса при запросе из 3 терминов (частоты терминов известны). Рассчитайте, сколько документов придется просмотреть, и сравните с последовательным сканированием.

### Билет №16
1.	Процесс работы поисковых роботов.
2.	Латентно-семантическое индексирование в задаче поиска.
3.	Задан запрос. Напишите псевдокод алгоритма обработки этого запроса. Предусмотреть исправление опечаток. Указать все компоненты и их последовательность.

### Билет №17
1.	Стоп-слова и их обработка в поиске.
2.	Подходы машинного обучения к задаче персонализации выдачи.
3.	Для трёх пользователей с заданной историей запросов предложите персонализированное переранжирование для заданного запроса. На основе их профилей измените порядок выдачи из 10 документов и объясните изменения.

### Билет №18
1.	Технология адаптивного персонализированного поиска.
2.	Соотношение поиска и рекомендаций в персонализированном поиске.
3.	Оценить, какие признаки важны для ранжирования заданного набора из 1000 документов и 100 запросов. Определите, какие признаки стоит удалить, и обоснуйте решение.

### Билет №19
1.	Применение архитектур Transformer и BERT для семантической интерпретации поисковых запросов.
2.	Референциальные системы оценивания значимости и их отличие от ссылочного ранжирования.
3.	Для заданного набора из 10 документов и 5 запросов рассчитайте оценки BM25 с заданными параметрами. Отсортируйте документы по убыванию релевантности для одного запроса и объясните, почему конкретный документ получил максимальный балл.

### Билет №20
1.	Неструктурированные и слабо структурированные данные в поисковом индексе.
2.	Специальные случаи поиска: узкоспециализированные запросы.
3.	Дан список из 100 документов на русском и английском языках. Предложите стратегию мультилингвистического поиска без перевода всех документов на один язык. Опишите, какие модели и индексы потребуются.
