## Текст теста

### Задание 1. (6 баллов)

Дан запрос пользователя: «как исправить ошибку 0x80070005 при обновлении виндовс 10».

Опишите алгоритмически, как система должна:

- а) исправить возможные опечатки (какой механизм: символьные n-граммы, нейросетевой spell-checker?);
- б) выявить сущности (error code, OS version, action);
- в) определить намерение пользователя (troubleshooting / how-to / download);
- г) расширить запрос синонимами и связанными терминами (какая модель для этого подойдёт?).


### Задание 2. (4 балла)

Приведите 3 примера запросов, где наивный BM25-поиск выдаст нерелевантные результаты, а нейросетевая семантическая модель — релевантные. Объясните, какую лингвистическую проблему в каждом случае решает нейросеть (синонимия, полисемия, синтаксическая вариативность).


## Критерии оценивания

## Эталон выполнения

**Задание 1:**

- а) **Символьные n-граммы + нейросетевой spell-checker:** символьные модели (например, на основе CharCNN или BPE) позволяют исправить «виндовс» → «windows», а нейросетевой корректор (T5 fine-tuned) обрабатывает контекстные ошибки.
- б) **Выявление сущностей:** NER-модель (BERT-based) выделяет: error_code = «0x80070005», os_version = «Windows 10», action = «обновление».
- в) **Определение намерения:** классификатор интентов (fine-tuned BERT) определяет класс troubleshooting.
- г) **Расширение запроса:** модель doc2query или T5 генерирует синонимы и связанные термины: «ошибка обновления», «код ошибки 0x80070005 решение», «Windows 10 update error fix».

**Задание 2:**

| Запрос | Проблема | Решение нейросети |
| :--- | :--- | :--- |
| «яблоко фрукт» | Полисемия (яблоко — фрукт или компания) | BERT учитывает контекст «фрукт» |
| «купить машину» | Синонимия («машина» = «автомобиль») | Эмбеддинги схожи по смыслу |
| «где уши у иголки» | Синтаксическая вариативность | BERT понимает смысл фразеологизма |
