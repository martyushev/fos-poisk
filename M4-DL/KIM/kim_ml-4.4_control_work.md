### Задача 1. Архитектуры (2 балла)

Даны две поисковые системы:

- Система А использует Bi-Encoder на базе BERT.
- Система В использует Cross-Encoder на базе BERT.

Ответьте на вопросы:

a) Какая система обеспечит меньший latency при обработке 10 000 запросов в секунду к коллекции из 10 млн документов? Обоснуйте.

b) Какая система даст более высокое качество ранжирования (NDCG)? Обоснуйте.

c) Предложите гибридную архитектуру, сочетающую преимущества обеих.

---

**Эталон ответа:**

a) **Bi-Encoder (система А)**. Эмбеддинги документов пре-вычисляются и сохраняются в FAISS индексе. Поиск выполняется через ANN за O(log N). Cross-Encoder требует прогона через сеть для каждой пары (запрос, документ), что даёт O(N · L), где L — latency одной инференс-операции.

b) **Cross-Encoder (система В)**. Модель видит совместный контекст (запрос + документ) и может вычислить тонкую релевантность на уровне токенов. Качество выше на 5–15% NDCG@10.

c) **Двухэтапная схема**: Bi-Encoder для candidate generation (быстрый поиск, топ-100), Cross-Encoder для reranking (точное ранжирование только кандидатов).

### Задача 2. Понимание запроса (3 балла)

Запрос пользователя: «яблоко падает»

a) Приведите минимум 3 различных намерения, которые может иметь пользователь.

b) Как BERT помогает разрешить полисемию в этом запросе? Опишите механизм на уровне attention-голов.

c) Предложите способ явного извлечения намерения (классификация) и покажите, как это изменит ранжирование.

---

**Эталон ответа:**

a) **Намерения:**
   1. Физика/наука: закон всемирного тяготения, эксперимент Ньютона.
   2. Садоводство: сбор урожая, падение яблок с дерева.
   3. Криптовалюты/инвестиции: падение цены токена Apple (или просто «яблоко» как актив).

b) **BERT** использует multi-head attention для учёта контекста. В каждом attention-слое вычисляются веса между токеном «яблоко» и окружающими словами. Если в запросе есть «падение» и нет других уточнений, BERT распределяет внимание между всеми словами, но для разрешения полисемии критичны контекстные слова. Однако одного слова недостаточно — нужны дополнительные сигналы.

c) **Извлечение намерения:** Fine-tuned BERT-base с классификацией на 5–10 классов. Выход CLS-токена подаётся в линейный слой с softmax. Ранжирование изменяется: для каждого класса намерения можно использовать разные веса признаков (например, для «физика» — больше эмбеддингов научных статей, для «садоводство» — больше сельскохозяйственных документов).

### Задача 3. Метрики (2 балла)

Даны 5 документов, ранжированных системой. Релевантность (по экспертной оценке): [1, 0, 1, 1, 0].

Вычислите:

a) Precision@3 и Recall@3;

b) NDCG@5 (с использованием логарифмического дисконта).

---

**Эталон ответа:**

a) **Precision@3** = (1 + 0 + 1) / 3 = 2/3 ≈ 0.67

   **Recall@3** = (1 + 0 + 1) / (1 + 1 + 1) = 2/3 ≈ 0.67

b) **NDCG@5:**

   Идеальный порядок релевантностей: [1, 1, 1, 0, 0]

   DCG = 1/1 + 0/log2(3) + 1/log2(4) + 1/log2(5) + 0/log2(6) = 1 + 0 + 0.5 + 0.431 + 0 = 1.931

   IDCG = 1/1 + 1/log2(3) + 1/log2(4) + 0 + 0 = 1 + 0.631 + 0.5 = 2.131

   NDCG@5 = 1.931 / 2.131 ≈ 0.906

### Задача 4. Генеративное переформулирование (3 балла)

Исходный запрос: «почему у меня не работает принтер epson l3250 после обновления win11»

a) Предложите 2 варианта переформулирования через генеративную модель (T5/BART).

b) Какие риски могут возникнуть при таком переформулировании?

c) Предложите способ валидации качества переформулирования (автоматическая метрика и/или human-in-the-loop).

---

**Эталон ответа:**

a) **Варианты переформулирования:**
   1. «Epson L3250 не работает после обновления Windows 11 — решение проблемы»
   2. «принтер Epson L3250 ошибка совместимости Windows 11 драйвер»

b) **Риски:**
   - Галлюцинации: модель может добавить несуществующие ошибки («код 0x00000d»).
   - Потеря намерения: замена «не работает» на «не печатает» может сузить проблему.
   - Семантический дрейф: изменение вопроса на «как обновить драйвер», хотя пользователь спрашивает о причине.

c) **Валидация:**
   - Автоматическая: BERTScore или семантическое сходство между исходным и переформулированным запросом (cosine similarity эмбеддингов).
   - Human-in-the-loop: A/B-тест, где эксперты оценивают качество переформулирования по шкале 1–5 (сохранение намерения, точность терминов, полезность для поиска).
