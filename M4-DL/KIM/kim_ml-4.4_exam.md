**Тип контроля:** итоговый

**Форма:** Экзамен / Защита проектов

**Максимальный балл:** 25 (интегрируется в общий вес модуля)

---

## Теоретические вопросы к экзамену

*Студент получает 1 вопрос + 1 практическую задачу на билете.*

1. Архитектура Transformer: multi-head attention, positional encoding, layer normalization. Почему это основа современных IR-систем?

2. BERT: предобучение (MLM + NSP), архитектурные особенности, применение в поиске.

3. Bi-Encoder vs Cross-Encoder: принципы работы, trade-off качество/latency, сценарии применения.

4. Векторный поиск: эмбеддинги, ANN-алгоритмы (HNSW, IVF), библиотеки (FAISS, ScaNN).

5. Метрики качества IR: Precision, Recall, F1, MAP, MRR, NDCG. Формулы и интерпретация.

6. Query Understanding: spell correction, NER для запросов, intent classification, query expansion.

7. Генеративные модели в поиске: doc2query, query rewriting, гибридный поиск (dense + sparse + generative).

8. Проблема domain adaptation: fine-tuning поисковых моделей на вертикальных доменах (медицина, юриспруденция, e-commerce).

## Практические задачи на экзамене

1. Рассчитать метрики NDCG@k, MRR, MAP по заданным ранжированным спискам.

2. Спроектировать архитектуру поисковой системы для заданного домена (выбор моделей, схема индексации, pipeline запроса).

3. Написать псевдокод fine-tuning BERT для задачи классификации intents.

4. Проанализировать ошибки нейросетевого ранжировщика на приведённых примерах и предложить способы их устранения.

## Защита лабораторных проектов

Студент демонстрирует работающий pipeline и отвечает на вопросы:

- Почему выбрана именно эта модель?
- Какие метрики достигнуты и почему именно они?
- Где узкие места системы (latency, память, качество)?
- Как система поведёт себя на out-of-domain запросах?
