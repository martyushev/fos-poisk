# Ключи для преподавателя (краткие)

## Контрольная работа — Задача 1

**a)** Bi-Encoder: эмбеддинги документов пре-вычисляются, поиск — ANN в FAISS, latency O(log N). Cross-Encoder требует прогона через сеть для каждой пары, latency O(N · L).

**б)** Cross-Encoder: видит совместный контекст (query + doc), качество выше на 5–15% NDCG.

**в)** Двухэтапная схема: Bi-Encoder для candidate generation (top-100), Cross-Encoder для reranking.

---

## Контрольная работа — Задача 2

**а)** Физика, садоводство, криптовалюты/инвестиции.

**б)** BERT через multi-head attention учитывает окружение. При недостатке контекста — неопределённость, требуется дообучение на доменных данных.

**в)** Fine-tuned BERT-base с классификацией intents через CLS-токен. Ранжирование адаптируется под класс намерения (разные веса признаков).

---

## Контрольная работа — Задача 3

**a)** Precision@3 = 2/3 ≈ 0.67; Recall@3 = 2/3 ≈ 0.67.

**б)** NDCG@5 ≈ 0.906.

*Примечание:* Если студент округляет log2(5) до 2.32, а log2(3) до 1.58, допустимым считается ответ в диапазоне 0.90–0.91. Главное требование — показать правильный идеальный порядок [1, 1, 1, 0, 0] и корректные знаменатели log2(i+1) для каждой позиции.
