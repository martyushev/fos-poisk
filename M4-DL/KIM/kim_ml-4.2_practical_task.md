## Часть А. Алгоритмическое проектирование (10 мин)

*1 задание × 5 баллов = 5 баллов*

### Задание 1

Дан запрос пользователя: «как исправить ошибку 0x80070005 при обновлении виндовс 10».

Опишите алгоритмически, как система должна:

1. исправить возможные опечатки (какой механизм: символьные n-граммы, нейросетевой spell-checker?);
2. выявить сущности (error code, OS version, action);
3. определить намерение пользователя (troubleshooting / how-to / download);
4. расширить запрос синонимами и связанными терминами (какая модель для этого подойдёт?).

---

**Эталонный ответ:**

1. **Spell correction:** Применяется комбинированный подход — символьные n-граммы (3–4 граммы) для быстрой коррекции опечаток на уровне символов («виндовс» → «windows») + нейросетевой spell-checker на основе BERT с маскированием, который исправляет контекстные ошибки. Приоритет отдаётся сохранению числовых кодов (0x80070005) без изменений.

2. **NER (Named Entity Recognition):** Используется fine-tuned BERT-based NER модель с кастомными тегами:
   - `ERROR_CODE`: 0x80070005
   - `OS_VERSION`: windows 10
   - `ACTION`: обновление (update)
   - `PROBLEM`: ошибка (error)

3. **Intent classification:** Fine-tuned BERT-base на датасете с классами намерений. Для данного запроса определяются два намерения:
   - Основное: `troubleshooting` (устранение неисправности) — 0.85
   - Вспомогательное: `how-to` (инструкция) — 0.15

4. **Query expansion:** Используется модель doc2query (T5-based) для генерации связанных запросов. Примеры расширений:
   - «исправить ошибку обновления windows 10 код 0x80070005»
   - «windows update error 0x80070005 fix»
   - «как решить проблему с обновлением win10»
   Дополнительно применяется синонимическое расширение через WordNet + эмбеддинги.

## Часть Б. Анализ семантических проблем (10 мин)

*1 задание × 5 баллов = 5 баллов*

### Задание 2

Приведите 3 примера запросов, где наивный BM25-поиск выдаст нерелевантные результаты, а нейросетевая семантическая модель — релевантные. Объясните, какую лингвистическую проблему в каждом случае решает нейросеть.

---

**Эталонный ответ:**

| № | Запрос | Проблема | Решение нейросети |
| :--- | :--- | :--- | :--- |
| 1 | «яблочный пирог рецепт» | Синонимия: BM25 ищет точное вхождение «яблочный пирог», не найдёт «apple pie» на англоязычных сайтах | Multilingual эмбеддинги (LaBSE) помещают семантически близкие фразы рядом в векторном пространстве независимо от языка |
| 2 | «банк» (в контексте финансов) | Полисемия: BM25 не различает «банк» (финансовое учреждение) и «банк» (скамейка/резервуар) | Контекстуальные эмбеддинги BERT учитывают соседние слова, определяя смысл по контексту запроса |
| 3 | «как починить звук на ноутбуке» | Синтаксическая вариативность: BM25 чувствителен к словоформам и порядку слов; эффективный документ может содержать «ноутбук звук не работает решение» | Cross-Encoder/SBERT понимают семантику предложения в целом, игнорируя поверхностные синтаксические различия |
