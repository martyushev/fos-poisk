## Тема

Реализация персонализированного поиска на основе поведенческих признаков

---

## Цель работы

Реализовать пайплайн персонализированного поиска, объединяющий:

- классический текстовый поиск (BM25);
- семантический поиск (эмбеддинги);
- поведенческие признаки пользователя.

---

## Ход выполнения работы

### Блок 1. Подготовка данных и симуляция поведения (1 час)

1. **Загрузка данных:** Использовать новостной корпус (например, из ЛР №3) или синтетический набор из 1000 документов.

2. **Симуляция пользователей:**
   - Создать класс `UserSimulator`.
   - Сгенерировать историю поиска для 5 пользователей с разными интересами (спорт, технологии, кулинария).
   - Сохранить логи по 10 запросов на пользователя.

### Блок 2. Инженерия поведенческих признаков (1 час)

1. **Признак `user_clicks_history`:**
   - Для каждого документа и пользователя вычислить количество кликов.
   - Применить **временной decay**:

   $$decayed\_score = \sum_{i} w_i \cdot e^{-\lambda \cdot age_i}$$

2. **Гибридный скор:**

   $$Score = \alpha \cdot S_{BM25} + (1-\alpha) \cdot (S_{emb} + S_{decay})$$

   Подобрать $\alpha$ экспериментально.

### Блок 3. Пост-процессинг (1 час)

1. **Фильтр безопасности:** Документы с `is_confidential=True` не показывать без токена.
2. **Бустинг новых документов:** Добавить бонус к скору для документов, добавленных за последний час.

### Блок 4. Оценка и анализ (1 час)

1. Для каждого пользователя выполнить запрос и вывести топ-5 результатов.
2. Сравнить выдачу для разных пользователей на один и тот же запрос.
3. Замерить время инференса (< 200 мс).
