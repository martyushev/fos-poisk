## Условия выполнения

**Формат работы:** Индивидуальная письменная работа.

**Регламент:** 30 минут.

**Инструкция для студента:** Выберите 3 этические проблемы, дайте развёрнутые ответы по каждой из них. Обосновывайте свои решения.


## Текст задания

### Анализ этических проблем поисковых систем

Вы — ML Researcher в компании, разрабатывающей поисковую систему нового поколения. Перед релизом системы необходимо провести этический аудит.

**Часть 1 (5 баллов).** Выберите 3 этические проблемы, которые могут возникнуть в вашей поисковой системе. Для каждой проблемы опишите:
- В чём заключается проблема;
- Кто может пострадать;
- Какие сценарии использования наиболее уязвимы.

**Часть 2 (5 баллов).** Предложите методы выявления и измерения каждой из выбранных проблем. Какие метрики и инструменты можно использовать?

**Часть 3 (5 баллов).** Предложите способы смягчения или устранения каждой проблемы. Опишите архитектурные или алгоритмические изменения, которые позволят сделать систему более справедливой и прозрачной.

## Эталон выполнения

**Часть 1. Выбор проблем:**

| Проблема | Суть | Пострадавшие | Уязвимые сценарии |
| :--- | :--- | :--- | :--- |
| **Bias по полу/расе** | Модель показывает стереотипные результаты | Маргинализированные группы | Поиск профессий, изображений людей |
| **Информационный пузырь** | Пользователь видит только подтверждающий контент | Все пользователи | Новостной поиск, рекомендации |
| **Нарушение конфиденциальности** | Утечка личных данных | Пользователи | Поиск медицинской информации |

**Часть 2. Методы выявления:**
- **Bias:** аудит датасетов на репрезентативность, fairness-метрики (demographic parity, equal opportunity), тестирование на сбалансированных тестовых наборах.
- **Информационный пузырь:** метрики diversity@k, novelty@k, покрытие категорий, анализ поведения пользователей в A/B-тестах.
- **Конфиденциальность:** аудит логов на наличие PII, дифференциальная приватность при обучении, анонимизация данных.

**Часть 3. Способы смягчения:**
- **Bias:** использование сбалансированных датасетов, наложение fairness-регуляризации при обучении, пост-процессинг результатов.
- **Информационный пузырь:** механизмы exploration (отображение части случайных результатов), баланс релевантности и разнообразия, умное кэширование.
- **Конфиденциальность:** федеративное обучение, шифрование данных, хранение только агрегированных профилей.
