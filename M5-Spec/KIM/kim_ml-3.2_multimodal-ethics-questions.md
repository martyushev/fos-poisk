## Условия выполнения

**Формат работы:** Устный опрос на семинарском занятии.

**Регламент:** Студент отвечает на 2–3 вопроса по выбору преподавателя. Время подготовки — 2 минуты на вопрос.

**Инструкция для студента:** Дайте развёрнутый ответ на поставленный вопрос. Используйте примеры для иллюстрации своих мыслей.


## Текст теста

**Вопросы для устного опроса:**

1. Что такое мультимодальный поиск и какие модальности он включает? Приведите примеры практических приложений.

2. Как осуществляется поиск по изображениям? Опишите основные подходы: CBIR, deep visual embeddings, мультимодальные модели.

3. Какие технологии лежат в основе голосового поиска? Опишите пайплайн обработки голосового запроса.

4. В чём особенности поиска в корпоративных данных? Какие типы данных и источники характерны для корпоративного поиска?

5. Какие вызовы возникают при индексации и ранжировании корпоративных документов разных форматов (PDF, электронные письма, чаты, презентации)?

6. Какие этические проблемы существуют в области информационного поиска? Назовите ключевые проблемы и методы их решения.

7. Что такое bias (смещение) в поисковых системах? Приведите примеры и объясните, как они возникают.

8. Как обеспечить конфиденциальность данных пользователей при персонализации поиска? Какие подходы используются для анонимизации данных?

9. Что такое информационная справедливость (information fairness) в поисковых системах? Приведите примеры её нарушения.

10. Как оценивать прозрачность и объяснимость ранжирования в поисковых системах? Для чего это необходимо?

## Эталон выполнения

**Вопрос 1:** Мультимодальный поиск — поиск по данным разных модальностей (текст, изображения, аудио, видео). Примеры: поиск по картинке в Google Lens, голосовой поиск в Яндекс.Станции, поиск видео по описанию.

**Вопрос 7:** Bias — систематическое смещение результатов, возникающее из-за нерепрезентативных данных обучения, алгоритмических ограничений или человеческих предубеждений. Пример: поиск по запросу «CEO» может показывать в основном мужчин, если данные обучения содержат такое смещение.

**Вопрос 10:** Прозрачность ранжирования — способность объяснить, почему документ оказался на конкретной позиции. Методы: SHAP, LIME для feature attribution, визуализация весов внимания в трансформерах, предоставление пользователю объяснений.
