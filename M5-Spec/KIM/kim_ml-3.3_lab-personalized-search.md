## Условия выполнения

**Инструментарий:** Python, Jupyter Notebook, библиотеки pandas, numpy, scikit-learn, lightgbm, matplotlib, seaborn.

**Данные:** Логи поиска (query, document, clicks, position, user_id).

**Задание:**

1. **Подготовка данных (4 балла):**
   - Загрузить логи поиска (query, document, clicks, position, user_id).
   - Сгруппировать данные по пользователям и запросам.
   - Построить профили пользователей на основе истории взаимодействий.
   - Создать признаки для персонализации: предпочтения пользователя (категории, частота кликов).

2. **Генерация признаков (6 баллов):**
   - Текстовые признаки: сходство запроса и документа (BM25/TF-IDF).
   - Поведенческие признаки: CTR, dwell time, история кликов.
   - Персонализированные признаки: сходство запроса с историей пользователя, релевантность категорий, оценка пользовательских предпочтений.

3. **Обучение модели персонализированного ранжирования (6 баллов):**
   - Обучить модель ранжирования с учётом персонализированных признаков (LGBMRanker).
   - Сравнить качество модели с персонализацией и без неё.
   - Замерить метрики: NDCG@5, MRR@10, MAP.

4. **Анализ персонализации (4 балла):**
   - Проанализировать, для каких групп пользователей персонализация даёт наибольший прирост качества.
   - Выявить типы запросов, где персонализация помогает, и где — вредит.

5. **Адаптация выдачи (6 баллов):**
   - Реализовать адаптацию выдачи в зависимости от контекста пользователя (геолокация, устройство, время суток).
   - Оценить влияние контекстных факторов на качество ранжирования.

6. **Оценка качества (4 балла):**
   - Провести офлайн-оценку персонализированной модели.
   - Сравнить метрики персонализированной модели с baseline (неперсонализированной).
   - Подготовить рекомендации по A/B-тестированию персонализации.

## Эталонные ответы на вопросы к защите

**1. Какие подходы к персонализации поиска вы знаете?**
- **Явная персонализация:** пользователь указывает предпочтения (жанры, категории).
- **Неявная персонализация:** на основе поведения (клики, просмотры, время пребывания).
- **Гибридная:** комбинация явных и неявных сигналов.
- **Контекстная:** учёт времени, геолокации, устройства.
- В работе реализован гибридный подход с использованием поведенческих сигналов.

**2. Какие признаки оказались наиболее значимыми?**
- Сходство запроса с историей пользователя.
- Частота взаимодействия с категориями документов.
- CTR и dwell time для конкретных типов документов.

**3. В каких случаях персонализация может ухудшить качество?**
- Для новых пользователей (холодный старт).
- Для информационных запросов, где объективность важнее предпочтений.
- При переобучении на исторические данные, если поведение изменилось.

**4. Как оценить качество онлайн?**
- A/B-тест: контрольная группа (без персонализации) против экспериментальной.
- Ключевые метрики: CTR, конверсия, retention, время на сайте.

**5. Как решается проблема холодного старта?**
- Использование контентных признаков вместо поведенческих.
- Коллективная фильтрация от похожих пользователей.
- Баланс между exploration и exploitation.
