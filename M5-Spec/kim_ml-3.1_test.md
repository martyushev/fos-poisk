### Вопрос 1

В системе персонализированного поиска используется модель ранжирования:

$$Score(q,d,u)=\alpha S_{BM25}(q,d)+\beta S_{emb}(q,d)+\gamma P(d,u)$$

где:

- $S_{BM25}$ — классическая текстовая релевантность;
- $S_{emb}$ — близость эмбеддингов;
- $P(d,u)$ — пользовательский профиль.

Какой параметр отвечает за влияние истории поведения пользователя?

- [ ] a) α
- [ ] b) β
- [x] c) γ
- [ ] d) размерность эмбеддинга

## Часть B. Верно / Неверно с объяснением

**3 вопроса × 1 балл (ответ + аргумент) = 3 балла**

### Вопрос 4

«Перевод запроса через машинный переводчик перед поиском всегда обеспечивает качество выше, чем прямой multilingual retrieval».

- [ ] Верно
- [x] Неверно

**Обоснование:** Машинный перевод может потерять смысловые оттенки, редкие термины и доменную специфику. Multilingual retrieval работает непосредственно в общем семантическом пространстве.

---

### Вопрос 5

«Использование истории кликов гарантирует повышение CTR и качества поиска».

- [ ] Верно
- [x] Неверно

**Обоснование:** История кликов может усилить filter bubble, закрепить прошлые ошибки и снизить diversity выдачи. Необходим контроль NDCG, diversity и novelty.

---

### Вопрос 6

«В мультимодальном поиске веса разных модальностей могут вычисляться динамически через gating-модель».

- [x] Верно
- [ ] Неверно

**Обоснование:** Веса могут зависеть от контекста запроса: $Score = w_t(q)S_{text} + w_i(q)S_{image}$, где $w_t$ и $w_i$ вычисляются нейросетью.

## Часть C. Практическое мини-задание

**1 вопрос × 4 балла = 4 балла**

### Ситуация

В корпоративной поисковой системе используется:

- BM25;
- multilingual embeddings;
- история пользователя.

После внедрения персонализации получены результаты:

| Метрика | До | После |
| :--- | :---: | :---: |
| CTR | 12% | 16% |
| NDCG@10 | 0.71 | 0.65 |
| Diversity | 0.55 | 0.32 |

### Задание

1. Почему рост CTR не означает улучшение качества поиска?
2. Какие изменения в модели ранжирования необходимо предложить?
3. Какие дополнительные эксперименты провести?

### Эталон ответа

1. Рост CTR может быть следствием «фильтрующего пузыря» — система показывает только то, что пользователь уже любит, снижая релевантность для новых запросов.
2. Необходимо ввести регуляризацию на разнообразие, добавить признаки novelty, пересмотреть веса для новых документов.
3. Провести A/B-тест с контрольной группой без персонализации, оценить долгосрочное удержание пользователей (retention).
