## Тема эссе

**«Как генеративные модели (T5, BART, GPT) могут использоваться для query rewriting в поисковых системах?»**

## Эталонный ответ (для преподавателя)

Генеративные модели (T5, BART, GPT) применяются в query rewriting для улучшения качества поиска через преобразование пользовательских запросов.

**Сценарии применения:**

1. **Коррекция опечаток и грамматических ошибок:** T5 fine-tuned на парах (неправильный запрос → правильный). Пример: «как печатать в ворде без мыши» → «как печатать в Microsoft Word без мыши».

2. **Расширение запроса (query expansion):** Генерация нескольких вариантов запроса для покрытия разных формулировок. Пример: «ошибка 404» → «error 404 page not found», «страница не найдена», «как исправить 404».

3. **Адаптация под поисковый движок:** Преобразование разговорных запросов в стиль, оптимальный для BM25. Пример: «где купить дешёвый ноутбук в москве» → «купить ноутбук цена Москва».

**Риски:**

- **Галлюцинации:** Модель может добавить несуществующие термины, искажая намерение.
- **Потеря намерения:** Переформулирование может изменить смысл запроса (например, «как удалить вирус» → «как установить антивирус»).
- **Latency:** Генерация занимает 100–500 мс, что критично для real-time поиска; требуется кэширование или асинхронная обработка.
