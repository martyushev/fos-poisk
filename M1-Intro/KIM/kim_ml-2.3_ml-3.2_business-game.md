**Индикаторы:** ML‑2.3, ML‑3.2  
**Средство оценивания:** Деловая игра «Архитектор баланса: от несбалансированных данных к надежному ИИ»  
**Форма контроля:** Командная работа + защита перед «заказчиком»  
**Max балл:** 20

## Легенда

Игроки — ведущие исследователи R&D‑отдела компании, разрабатывающей систему предиктивного технического обслуживания (*Predictive Maintenance*) для критической инфраструктуры (авиационные двигатели / турбины ГЭС).

Заказчик предоставил исторические данные телеметрии за 5 лет. Система должна предсказывать отказ узла (класс Failure) за 48 часов до события.  
**Проблема:** класс Failure составляет 0.8% выборки (сильный дисбаланс). Accuracy = 99%, но система бесполезна — предсказывает только Normal. До релиза 2 недели.

## Ход игры (4 академических часа)

| Этап | Время | Задача |
| :--- | :--- | :--- |
| **0. Брифинг** | 20 мин | Получение данных (`train.csv`, `test.csv`), бизнес‑требований (FPR < 5%, Recall > 95%), ограничений (инференс < 50 мс). EDA. |
| **1. Базлайн** | 40 мин | Обучение простой модели без балансировки. Расчёт Accuracy, Precision/Recall/F1, Confusion Matrix. Формулировка гипотезы о провале. |
| **2. Хакатон ресемплинга** | 60 мин | Реализация ≥3 подходов: over‑sampling (SMOTE, ADASYN), under‑sampling (RandomUnderSampler, Tomek Links), комбинированные (SMOTETomek), синтетические методы. Оценка F‑beta (β=2). |
| **3. Архитектурные решения** | 60 мин | Использование `class_weight` / `scale_pos_weight`, переход к устойчивым алгоритмам (CatBoost/LGBM, BalancedBaggingClassifier), пороговые трансформации (Platt Scaling, Isotonic Regression). Ранжирование по Pareto‑фронту. |
| **4. Оценка и защита** | 45 мин | Выбор финального пайплайна, слепое тестирование, Precision‑Recall кривая, AUPRC, Cost Matrix. Подготовка отчёта. |
| **5. Презентация** | 20 мин | Защита перед заказчиком (3 мин + 2 мин Q&A). Доказательство Recall > 95% и экономической обоснованности. |
