## Задание

Набор данных `search_eval.csv` содержит результаты работы двух алгоритмов ранжирования (`model_baseline` — классический BM25, `model_ai` — нейросетевой рангер) для 1000 поисковых запросов.

**Структура файла:**
- `query_id` — идентификатор запроса;
- `doc_id` — идентификатор документа;
- `score_bm25` — оценка релевантности от классической модели;
- `score_nn` — оценка релевантности от AI‑модели;
- `label` — истинная релевантность (2 — высокорелевантный, 1 — частично релевантный, 0 — нерелевантный).

Распределение `label`: ~85% — 0, ~10% — 1, ~5% — 2.

### Задачи

1. **Анализ распределения.** Построить гистограмму распределения меток `label`. Рассчитать коэффициент Джини или энтропию для подтверждения степени дисбаланса. *(2 балла)*

2. **Базовая оценка.** Отсортировать документы для каждого запроса по убыванию `score_bm25` и `score_nn`. Рассчитать для обеих моделей метрики: MAP@10, nDCG@10, Precision@1, Recall@100. *(4 балла)*

3. **Выявление проблемы.** Проанализировать полученные значения. Продемонстрировать численно, как высокий Recall@100 при низком Precision@1 у базовой модели маскирует её неспособность выводить релевантные документы в топ выдачи. *(3 балла)*

4. **Борьба с дисбалансом (Sampling & Weighting):**
   - реализовать перевыборку (*upsampling*) класса `label=2` внутри каждого батча с коэффициентом `k=3`; *(2 балла)*
   - внедрить взвешенную версию nDCG, где gain для документов с `label=2` умножается на вес `α`, а с `label=1` — на `β` (`α > β > 1`). Например: `α ∈ {3, 5, 7}`, `β ∈ {1.5, 2.0, 2.5}`, исключая комбинации, где `α ≤ β`. *(2 балла)*

5. **Сравнение и выбор.** Пересчитать nDCG@10 для обеих моделей с учётом сбалансированной выборки и новых весов. Сравнить дельту прироста между моделями. Обосновать, какая модель демонстрирует лучшую обобщающую способность после коррекции дисбаланса. *(4 балла)*

6. **Отчётность.** Подготовить краткий аналитический отчёт (до 2 страниц), включающий графики распределений, таблицы исходных и скорректированных метрик, обоснование выбранных гиперпараметров (`α`, `β`), анализ вычислительной сложности методов балансировки и итоговый вывод о применимости моделей в production‑среде с учётом trade‑off между качеством и latency. *(3 балла)*
