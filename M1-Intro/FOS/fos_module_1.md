# Фонд оценочных средств (ФОС)
## Модуль 2. Машинное обучение в поиске

**Профессиональная роль:** ML Engineer / ML Researcher

**Объем модуля:** 12 академических часов (4 часа лекций + 8 часов лабораторный практикум)

---

## Назначение ФОС

Фонд оценочных средств предназначен для оценки уровня сформированности профессиональных компетенций обучающихся в области разработки и применения методов машинного обучения для задач поиска и ранжирования.

В рамках модуля оценивается способность обучающегося:

- различать подходы к обучению ранжированию (pointwise, pairwise, listwise);
- применять нейросетевые модели (DSSM, CDSSM) для семантического поиска;
- использовать предобученные эмбеддинги (Word2Vec, GloVe) для представления запросов и документов;
- реализовывать пайплайн Learning to Rank (LTR) от предобработки до оптимизации метрик;
- анализировать качество ранжирования и интерпретировать результаты.

## Описание оценочных средств

### 1. Тестирование по подходам к ранжированию (ML-2.1)

**Форма:** Письменный тест (10 вопросов с выбором ответа)  
**Время:** 15 минут  
**Темы:** Pointwise, pairwise, listwise подходы, их сравнение, области применения.

---

### 2. Деловая игра «Архитектурный пинг-понг» (ML-2.1)

**Форма:** Командная игра (3–4 человека)  
**Время:** 20 минут  
**Описание:** Команды по очереди предлагают архитектурные решения для задачи ранжирования, обосновывая выбор подхода (pointwise/pairwise/listwise). Противоположная команда выявляет риски и ограничения.

**Критерии оценки:**
| Критерий | Баллы |
| :--- | :--- |
| Обоснованность выбора подхода | 0–4 |
| Корректность выявленных рисков | 0–4 |
| Качество аргументации в дискуссии | 0–2 |

---

### 3. Эссе-миниатюра по эмбеддингам и семантическому поиску (ML-2.2)

**Форма:** Письменная работа  
**Время:** 15 минут  
**Тема:** «Сравнение статических (Word2Vec, GloVe) и контекстуальных эмбеддингов для задач семантического поиска. Роль DSSM в современном поиске.»

**Требования к эссе:**
- Объём: 200–250 слов.
- Сравнительный анализ двух подходов.
- Примеры применения в поисковых системах.
- Обсуждение ограничений.

**Критерии оценки:**
| Критерий | Баллы |
| :--- | :--- |
| Полнота сравнительного анализа | 0–5 |
| Качество примеров | 0–4 |
| Глубина анализа ограничений | 0–3 |
| Логика и аргументированность | 0–3 |

---

### 4. Лабораторная работа №2 «Реализация алгоритма ранжирования на основе ML» (ML-3.2)

**Тема:** Реализация пайплайна Learning to Rank с использованием классических методов ML (градиентный бустинг, логистическая регрессия) и сравнение с DSSM-подходом.

**Задание:**
1. Загрузить и предобработать данные (логи поисковых запросов, MS MARCO).
2. Сформировать признаки для LTR (текстовые, статистические, кросс-фичи).
3. Реализовать pointwise (регрессия/классификация) и pairwise (RankNet/RankSVM) подходы.
4. Реализовать эмбеддинг-представление запросов и документов (Word2Vec/GloVe).
5. Провести оценку качества с использованием NDCG@10, MAP@10, MRR@10.
6. Сравнить результаты и выбрать оптимальную модель.

**Требования к сдаче:**
- Jupyter Notebook с воспроизводимым кодом и комментариями.
- Таблицы с метриками.
- Графики сравнения моделей.
- Аналитический отчет (2–3 страницы).

**Критерии оценки:**
| Критерий | Баллы |
| :--- | :--- |
| Корректность реализации пайплайна | 0–10 |
| Качество генерации признаков и эмбеддингов | 0–8 |
| Правильность реализации подходов LTR | 0–10 |
| Полнота и корректность оценки качества | 0–6 |
| Качество аналитического вывода | 0–6 |
| **Итого** | **40** |

## Методические рекомендации

1. Все оценочные средства должны проводиться в соответствии с календарным планом.
2. Лабораторная работа выполняется в малых группах (2–3 человека) с последующей защитой.
3. Деловая игра проводится в формате командного соревнования с элементами дискуссии.
4. Для оценивания лабораторной работы используется чек-лист и рубрикатор.
5. Итоговый тест проводится в письменной или электронной форме.
6. Акцент в оценивании делается на практическое применение изученных подходов и понимание их ограничений.
