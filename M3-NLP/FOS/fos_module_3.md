# Модуль 3. NLP в поиске: обработка запросов и документов

**Профессиональная роль:** ML Engineer / NLP Specialist

---

## Назначение ФОС

Фонд оценочных средств предназначен для оценки уровня сформированности профессиональных компетенций обучающихся в области применения методов обработки естественного языка (NLP) для задач информационного поиска.

В рамках модуля оценивается способность обучающегося:

- применять методы предобработки текста (токенизация, лемматизация, удаление стоп-слов);
- использовать статистические модели (TF-IDF, LSA) для векторного представления текстов;
- кластеризовать текстовые документы и интерпретировать результаты;
- анализировать проблемы поисковых систем (стоп-слова, точные идентификаторы, мультимодальность);
- проектировать пайплайн обработки данных от краулинга до индексации.

### Критерии оценивания лабораторной работы

| Оценка | Уровень выполнения |
| :--- | :--- |
| Отлично | Пайплайн работает идеально. Проведена глубокая интерпретация результатов. Анализ ошибок демонстрирует понимание внутренних механизмов моделей (почему именно SVD/KMeans ошиблись). |
| Хорошо | Код рабочий, метрики посчитаны верно. Однако этап анализа может быть поверхностным (например, просто скопированы топ-слов без попытки объяснить их смысл) или есть мелкие недочеты в очистке текста. |
| Удовлетворительно | Базовые этапы выполнены (TF-IDF построен, кластеры найдены), но присутствуют технические ошибки (неправильный выбор k, игнорирование лемматизации, мусор в тексте) или отчет оформлен небрежно. |
| Неудовлетворительно | Код выдает критические ошибки при запуске, матрица признаков не построена, логика группировки данных нарушена или отсутствует блок выводов и анализа. |

---

### Итоговая формула оценивания курса

**Итог = 0.25 · Текущий контроль (тест + задачи) + 0.40 · Лабораторная работа + 0.35 · Итоговый тест**

## Методические рекомендации

1. Мини-тест проводится в начале занятия для актуализации знаний (5 минут).
2. Задачи решаются в малых группах с последующим обсуждением (10 минут).
3. Лабораторная работа выполняется индивидуально или в парах, защита проводится в форме собеседования.
4. Итоговый тест проводится в письменной форме, время выполнения — 40 минут.
5. При оценке лабораторной работы особое внимание уделяется этапу анализа ошибок кластеризации.
