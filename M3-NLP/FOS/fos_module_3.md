## Назначение ФОС

Фонд оценочных средств предназначен для оценки уровня сформированности профессиональных компетенций обучающихся в области применения методов обработки естественного языка (NLP) в поисковых системах, включая предобработку текстов, векторизацию, снижение размерности и кластеризацию документов.

В рамках модуля оценивается способность обучающегося:

- понимать роль NLP в интеллектуальном поиске;
- применять методы предобработки текста: токенизацию, лемматизацию, фильтрацию стоп-слов;
- строить векторные представления текстов с использованием TF-IDF;
- применять методы снижения размерности (LSA/SVD) для визуализации и кластеризации;
- использовать алгоритмы кластеризации (KMeans) для группировки семантически близких текстов;
- интерпретировать результаты кластеризации и анализировать ошибки модели;
- понимать проблемы и ограничения языковых моделей в поисковых системах;
- различать типы данных (структурированные, полуструктурированные, неструктурированные) и методы их индексации.

1. **Мини-тест** проводится в начале семинарского занятия (5 минут) для проверки понимания ключевых концепций NLP в поиске. Оценивается по количеству правильных ответов.

### Блок 3. Итоговый контроль

#### 3.1 Итоговый тест

Итоговый тест включает 15 вопросов различных типов. Время выполнения — 40 минут.

**Часть А. Вопросы с выбором одного ответа (12 вопросов × 2 балла = 24 балла)**

**1. Какова основная задача краулера (поискового робота) на этапе post-crawling (после загрузки страницы), но до попадания документа в индекс?**

а) Расчет PageRank для найденных ссылок.

б) Лингвистическая предобработка: токенизация, лемматизация, удаление разметки HTML и извлечение чистого текста.

в) Определение геолокации сервера-хоста.

г) Проверка SSL-сертификата сайта.


**2. В чем заключается главная проблема использования стоп-слов при построении классического инвертированного индекса?**

а) Они увеличивают вычислительную сложность алгоритма BM25.

б) Они раздувают размер словаря (лексикона) и списков позиций (posting lists), не добавляя семантической ценности запросу.

в) Стоп-слова невозможно корректно отлемматизировать.

г) Наличие стоп-слов всегда приводит к переобучению ML-моделей.


**3. Почему использование кросс-экодера (Cross-encoder BERT) напрямую для ранжирования миллиона документов технически нереализуемо в production-поиске?**

а) Кросс-экодеры не поддерживают русский язык без перевода.

б) Из-за квадратичной сложности механизма внимания O(n²) время обработки одного запроса превысит допустимые задержки latency.

в) Для работы кросс-экодера обязательно наличие графического процессора на стороне пользователя.

г) Кросс-экодер требует ручной разметки каждого документа асессорами.


**4. Что является основным отличием латентно-семантического индексирования (LSI/LSA) от обычного векторного представления TF-IDF?**

а) LSI работает только с изображениями, а TF-IDF — с текстом.

б) LSI использует сингулярное разложение (SVD) для выявления скрытых (латентных) связей между терминами через их совместную встречаемость в документах.

в) LSI полностью игнорирует частоту слов, опираясь только на порядок букв.

г) LSI требует наличия нейросетевого ускорителя.


**5. Какой подход наиболее эффективен для решения проблемы «холодного старта» (отсутствия данных о кликах) для абсолютно нового товара в интернет-магазине?**

а) Использование исторического CTR категории.

б) Применение признаков поведенческой коллаборативной фильтрации.

в) Использование контентных эмбеддингов (текст + изображение) и атрибутов товара для вычисления сходства с уже популярными товарами (Content-based filtering).

г) Принудительное размещение на первой позиции выдачи на 24 часа.


**6. При анализе поискового лога обнаружено, что пользователи массово переформулируют запрос «починить кран капает» в «услуги сантехника круглосуточно». Какой паттерн это выявляет?**

а) Проблема синонимии терминов.

б) Наличие скрытого интента (коммерческого намерения), который не покрывается исходным информационным запросом.

в) Ошибка токенизатора при обработке слова «капает».

г) Низкий авторитет доменов со статьями про ремонт.


**7. Какую роль играет анализ эмоций (Sentiment Analysis) в задаче информационного поиска (например, поиске отзывов или новостей)?**

а) Позволяет определить пол автора текста.

б) Используется исключительно для блокировки негативного контента.

в) Служит мощным признаком ранжирования: для запросов типа «лучшие наушники» документы с позитивной окраской должны иметь приоритет, а для "причины поломки Х" - нейтральную или негативную экспертизу.

г) Заменяет собой необходимость анализа тональности изображений.

**8. Что из перечисленного относится к слабо структурированным данным в контексте построения поискового индекса?**

а) Таблица SQL базы данных пользователей.

б) JSON-файл со строгой схемой категорий товаров.
