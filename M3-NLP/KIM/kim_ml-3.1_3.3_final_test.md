### Вопрос 2

В чем заключается главная проблема использования стоп-слов при построении классического инвертированного индекса?

- [ ] а) Они увеличивают вычислительную сложность алгоритма BM25.
- [x] б) Они раздувают размер словаря (лексикона) и списков позиций (*posting lists*), не добавляя семантической ценности запросу.
- [ ] в) Стоп-слова невозможно корректно отлемматизировать.
- [ ] г) Наличие стоп-слов всегда приводит к переобучению ML-моделей.

### Вопрос 3

Почему использование кросс-экодера (Cross-encoder BERT) напрямую для ранжирования миллиона документов технически нереализуемо в production-поиске?

- [ ] а) Кросс-экодеры не поддерживают русский язык без перевода.
- [x] б) Из-за квадратичной сложности механизма внимания O(n^2) время обработки одного запроса превысит допустимые задержки latency.
- [ ] в) Для работы кросс-экодера обязательно наличие графического процессора на стороне пользователя.
- [ ] г) Кросс-экодер требует ручной разметки каждого документа асессорами.

### Вопрос 4

Что является основным отличием латентно-семантического индексирования (LSI/LSA) от обычного векторного представления TF-IDF?

- [ ] а) LSI работает только с изображениями, а TF-IDF — с текстом.
- [x] б) LSI использует сингулярное разложение (SVD) для выявления скрытых (латентных) связей между терминами через их совместную встречаемость в документах.
- [ ] в) LSI полностью игнорирует частоту слов, опираясь только на порядок букв.
- [ ] г) LSI требует наличия нейросетевого ускорителя.

### Вопрос 5

Какой подход наиболее эффективен для решения проблемы «холодного старта» (отсутствия данных о кликах) для абсолютно нового товара в интернет-магазине?

- [ ] а) Использование исторического CTR категории.
- [ ] б) Применение признаков поведенческой коллаборативной фильтрации.
- [x] в) Использование контентных эмбеддингов (текст + изображение) и атрибутов товара для вычисления сходства с уже популярными товарами (Content-based filtering).
- [ ] г) Принудительное размещение на первой позиции выдачи на 24 часа.

### Вопрос 6

При анализе поискового лога обнаружено, что пользователи массово переформулируют запрос «починить кран капает» в «услуги сантехника круглосуточно». Какой паттерн это выявляет?

- [ ] а) Проблема синонимии терминов.
- [x] б) Наличие скрытого интента (коммерческого намерения), который не покрывается исходным информационным запросом.
- [ ] в) Ошибка токенизатора при обработке слова «капает».
- [ ] г) Низкий авторитет доменов со статьями про ремонт.

### Вопрос 7

Какую роль играет анализ эмоций (Sentiment Analysis) в задаче информационного поиска (например, поиске отзывов или новостей)?

- [ ] а) Позволяет определить пол автора текста.
- [ ] б) Используется исключительно для блокировки негативного контента.
- [x] в) Служит мощным признаком ранжирования: для запросов типа «лучшие наушники» документы с позитивной окраской должны иметь приоритет, а для «причины поломки Х» — нейтральную или негативную экспертизу.
- [ ] г) Заменяет собой необходимость анализа тональности изображений.

### Вопрос 9

Как адаптивный персонализированный поиск разрешает конфликт между высокой релевантностью документа глобально и его низкой ценностью для конкретного пользователя?

- [ ] а) Полностью отключает общие признаки релевантности.
- [x] б) Применяет взвешенную сумму скорингов: Global_Score × α + Personalization_Feature_Score × (1 - α), динамически подбирая коэффициент α.
- [ ] в) Удаляет непонравившиеся пользователю документы из глобальной базы навсегда.
- [ ] г) Показывает только те документы, которые пользователь открывал ранее.

### Вопрос 10

В чем ключевое отличие узкоспециализированного (вертикального) поиска от общего веб-поиска?

- [ ] а) Вертикальный поиск использует более мощные серверы.
- [x] б) В использовании специфических онтологий, фасетной навигации и обогащении фичей данными из профильных баз (например, химические формулы, каталожные номера запчастей).
- [ ] в) Вертикальный поиск не учитывает ссылочное ранжирование.
- [ ] г) Вертикальный поиск работает только внутри одной страны.

### Вопрос 11

Какая метрика лучше всего отражает удовлетворенность пользователя, если ему нужно найти ровно один верный ответ (например, номер телефона поддержки)?

- [ ] а) Mean Average Precision (MAP).
- [x] б) MRR (Mean Reciprocal Rank) — так как она сильно штрафует за нахождение ответа ниже первого места.
- [ ] в) Recall@100.
- [ ] г) F1-score текстовой классификации.

### Вопрос 15

Опишите алгоритм действий поисковой системы (три шага), когда пользователь вводит запрос, которому соответствует ноль документов в индексе (Zero Results Search), с целью удержать пользователя на сервисе.

**Эталонный ответ:**

1. **Анализ неудачи и автоматическое исправление:** Исправление опечаток и раскладки клавиатуры.
2. **Семантическое расширение (Fuzzy Matching):** Поиск ближайших соседей по векторным эмбеддингам документов (показ результатов со схожим смыслом при отсутствии точных лексических совпадений).
3. **Предложение альтернатив (Query Suggestion/Relaxation):** Предложение популярных похожих поисковых трендов («Did you mean?») или переход в категорию каталога с сохранением фильтров.
