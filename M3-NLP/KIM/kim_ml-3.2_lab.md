## Этапы выполнения

### Этап 1. Сбор и первичная очистка данных (Text Wrangling)

1. Загрузить коллекцию новостных статей (например, *20 Newsgroups*) или поисковый лог (*query–document*), содержащий не менее 5 000 документов.
2. Выполнить нормализацию текста (приведение к нижнему регистру, удаление дубликатов).
3. Очистить текст от HTML-тегов, URL-адресов, спецсимволов и лишних пробелов.
4. Провести токенизацию и лемматизацию (pymorphy2 для русского языка, nltk для английского).
5. Удалить стоп-слова. Зафиксировать в отчете изменение размера словаря.

### Этап 2. Векторное представление текста (Feature Extraction)

1. Построить матрицу терм-документ с помощью CountVectorizer, проанализировать разреженность.
2. Построить TF-IDF матрицу с параметрами `max_df=0.9`, `min_df=2`, `ngram_range=(1, 2)`.
3. Сохранить матрицу и список признаков.
4. Найти топ-5 слов с самым высоким IDF и топ-5 слов с самым высоким средним TF-IDF.

### Этап 3. Снижение размерности и визуализация

1. Применить TruncatedSVD (LSA) для снижения размерности до 50–100 компонент.
2. Использовать t-SNE или UMAP для перевода в 2D-пространство.
3. Построить scatter plot документов.

### Этап 4. Кластеризация семантически близких текстов

1. Обучить модель KMeans на TF-IDF векторах.
2. Определить оптимальное число кластеров (k) с помощью метода «локтя» и коэффициента силуэта.
3. Для каждого кластера вывести топ-10 ключевых слов через анализ центров кластеров.
4. Провести качественный анализ: сопоставить кластеры с реальными темами.
5. Выявить и проанализировать ошибки кластеризации (не менее 3 примеров документов, попавших в «чужой» кластер, с объяснением причин).

## Критерии оценивания

| Оценка | Уровень выполнения |
| :--- | :--- |
| Отлично | Пайплайн работает идеально. Проведена глубокая интерпретация результатов. Анализ ошибок демонстрирует понимание внутренних механизмов моделей (почему именно SVD/KMeans ошиблись). |
| Хорошо | Код рабочий, метрики посчитаны верно. Однако этап анализа может быть поверхностным (например, просто скопированы топ-слов без попытки объяснить их смысл) или есть мелкие недочеты в очистке текста. |
| Удовлетворительно | Базовые этапы выполнены (TF-IDF построен, кластеры найдены), но присутствуют технические ошибки (неправильный выбор k, игнорирование лемматизации, мусор в тексте) или отчет оформлен небрежно. |
| Неудовлетворительно | Код выдает критические ошибки при запуске, матрица признаков не построена, логика группировки данных нарушена или отсутствует блок выводов и анализа. |
