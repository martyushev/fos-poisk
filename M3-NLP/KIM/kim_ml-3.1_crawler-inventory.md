| № | Проверяемый элемент | Уровень сложности | Балл |

## Условия выполнения

**Формат работы:** Индивидуальная письменная работа.

**Регламент:** 10 минут.

**Инструкция для студента:** Распределите источники по типам (Неструктурированные / Полуструктурированные / Структурированные). Для каждого типа укажите главную сложность при создании единого поискового индекса.


## Текст задания

**«Инвентаризация краулера»**

**Контекст:** Вам нужно проиндексировать базу знаний крупной IT-компании. Она состоит из:

1. Внутренних Confluence-страниц (HTML + вложенные PDF).
2. Таблиц со списком серверов (JSON, поля: IP, owner, status).
3. Чатов техподдержки (неструктурированные потоки текста).

**Задание:** Распределите эти источники по типам (Неструктурированные / Полуструктурированные / Структурированные). Для каждого типа укажите главную сложность при создании единого поискового индекса.


## Критерии оценивания

| Критерий | Макс. балл | Описание |
| :--- | :--- | :--- |
| Правильная классификация | 3 | Каждый источник отнесён к верному типу |
| Обоснование сложности для каждого типа | 6 | Для каждого типа указана главная проблема индексации (по 2 балла за тип) |
| Четкость и структурированность | 1 | Ответ оформлен в виде таблицы или структурированного списка |


## Эталон выполнения

| Источник | Тип | Главная сложность |
| :--- | :--- | :--- |
| Confluence-страницы (HTML + PDF) | **Полуструктурированные** | Извлечение чистого текста из вложений (PDF) и сохранение иерархии заголовков для контекстного поиска; разная структура страниц требует гибкого парсинга |
| Таблицы со списком серверов (JSON) | **Структурированные** | Необходимость поддерживать точные значения полей (IP, status) без семантического размытия; важны числовые и категориальные фильтры |
| Чаты техподдержки | **Неструктурированные** | Диалоговый характер данных: важны не только отдельные сообщения, но и цепочки общения; сложно выделить ключевые сущности из-за разговорного языка, сленга и опечаток |

**Максимальный балл:** 10.
