# Методические указания обучающимся

## Дисциплина: Модели и методы интеллектуального поиска


### 1. Общие положения

Дисциплина «Модели и методы интеллектуального поиска» направлена на формирование у студентов теоретических знаний и практических навыков разработки, настройки и оценки интеллектуальных поисковых систем, использующих методы машинного обучения, обработки естественного языка, векторного поиска, больших языковых моделей и RAG-подходов.

В рамках дисциплины студенты осваивают:
- принципы построения современных интеллектуальных поисковых систем;
- методы и алгоритмы обработки естественного языка, применяемые в поиске;
- современные подходы NLP, эмбеддингов и семантического поиска;
- навыки проектирования и реализации RAG-сценариев;
- методы оценки качества поисковой выдачи.

**Профессиональная роль:** ML Researcher.

### 4. Подготовка к различным формам контроля

#### 4.1 Устный опрос

**Проверяемые навыки:** понимание теоретического материала, способность объяснять концепции, умение приводить примеры.

**Рекомендации по подготовке:**
1. Изучить материал лекции, осмысливая логику, а не механически запоминая определения.
2. Подготовить примеры для иллюстрации каждого теоретического положения.
3. Практиковать устное объяснение тем другим лицам.
4. Акцентировать внимание на причинно-следственных связях и ограничениях подходов.

**Типичные ошибки:**
- Заучивание фактов без понимания.
- Ответы без конкретных примеров.

#### 4.2 Мини-тест

**Проверяемые навыки:** знание ключевых понятий и фактов.

**Рекомендации по подготовке:**
1. Повторить основные определения и формулы.
2. Обратить внимание на ключевые различия между подходами.
3. Вспомнить архитектурные отличия моделей.

**Типичные ошибки:**
- Игнорирование терминологии.
- Невнимательное чтение условий вопросов.

#### 4.3 Практическое задание

**Проверяемые навыки:** применение знаний для решения конкретных задач, аналитические способности, умение обосновывать выбор методов.

**Рекомендации по подготовке:**
1. Внимательно прочитать условие, выделить ключевые требования.
2. Разработать план решения: выбор метода, модели, архитектуры с обоснованием.
3. Продумать способы оценки результата и метрики.

**Структура ответа:**
- Описание проблемы.
- Выбор метода с обоснованием.
- Описание реализации.
- Оценка результатов.
- Анализ ограничений.

#### 4.4 Эссе-миниатюра

**Проверяемые навыки:** способность кратко и аргументированно анализировать проблему, предлагать решения.

**Рекомендации по подготовке:**
1. Определить структуру: тезис → аргументы → вывод.
2. Использовать конкретные примеры.
3. Придерживаться ограничения по объёму (150–200 слов).
4. Проверить логическую связанность предложений.

**Типичные ошибки:**
- Общие фразы без конкретики.
- Отсутствие чёткой позиции.

#### 4.5 Лабораторная работа

**Проверяемые навыки:** практическое программирование, реализация пайплайнов, анализ результатов, формулирование выводов.

**Подготовка к выполнению:**
1. Изучить задание заранее, продумать архитектуру решения.
2. Освоить необходимые библиотеки (pandas, numpy, scikit-learn, lightgbm, transformers).
3. Подготовить план отчёта до начала выполнения.

**Подготовка к защите:**
1. Убедиться в воспроизводимости кода (фиксация версий, random seed).
2. Подготовить краткую презентацию результатов (3–5 слайдов).
3. Продумать ответы на вопросы преподавателя.
4. Быть готовым признать ограничения решения.

**Структура отчёта:**
1. Цель работы.
2. Описание данных (источник, объём, структура).
3. Методология (модели, методы, обоснование).
4. Реализация (основные этапы, ключевые фрагменты кода).
5. Результаты (метрики, графики, таблицы).
6. Анализ (интерпретация, ошибки, ограничения).
7. Выводы.

#### 4.6 Контрольная работа

**Проверяемые навыки:** комплексное понимание материала модулей 1–4, расчёт метрик, проектирование архитектур, анализ проблем.

**Рекомендации по подготовке:**
1. Повторить ключевые темы каждого модуля.
2. Практиковать расчёт метрик (NDCG, MAP, MRR, Precision, Recall), обращая внимание на формулу дисконта и идеальный порядок.
3. Продумать типовые архитектуры для разных задач.

**Типичные ошибки:**
- Пропуск промежуточных вычислений.
- Необоснованные утверждения.
- Арифметические ошибки.

#### 4.7 Экзамен

**Проверяемые навыки:** всестороннее владение материалом, способность рассуждать о выборе методов, понимание компромиссов и ограничений.

**Рекомендации по подготовке:**
1. Повторить теоретические вопросы из списка.
2. Практиковать решение задач.
3. Выстраивать ответ от общего к частному.
4. Объяснять не только «как», но и «почему».

**Структура ответа:**
1. Введение.
2. Основная часть с обоснованиями и примерами.
3. Заключение с ключевыми выводами.

**Типичные ошибки:**
- Перечисление фактов без связей.
- Ответы без примеров.
- Отсутствие уверенности.


### 5. Рекомендации по работе с модулями

#### 5.1 Модуль 1. Введение в интеллектуальный поиск

**Ключевые темы:**
- Архитектура поисковой системы.
- Инвертированный индекс.
- Классические модели ранжирования (TF-IDF, BM25).
- Факторы ссылочного ранжирования (PageRank, TrustRank).
- Метрики качества (MAP, nDCG, Precision, Recall).
- Проблема дисбаланса релевантности.

**Рекомендации:** понять механизм работы инвертированного индекса как основы поиска. Обратить внимание на отличие BM25 от TF-IDF. В лабораторной работе интерпретировать результаты.

**Типичные ошибки:** путаница между Precision и Recall; некорректный расчёт NDCG; непонимание ограничений классических моделей.

#### 5.2 Модуль 2. Машинное обучение в поиске

**Ключевые темы:**
- Pointwise, pairwise, listwise подходы.
- LambdaMART и оптимизация NDCG.
- Bi-Encoder и Cross-Encoder.
- DSSM, CDSSM.
- Инженерия признаков.
- Position Bias, холодный старт.

**Рекомендации:** разобраться в различиях подходов на уровне функции потерь. В лабораторной работе обеспечить правильную группировку по query_id.

**Типичные ошибки:** неправильная группировка данных; непонимание компромисса между качеством и задержкой; игнорирование Position Bias.

#### 5.3 Модуль 3. Обработка естественного языка для поисковых систем

**Ключевые темы:**
- Предобработка текста.
- TF-IDF и векторное пространство.
- LSA/LSI.
- Кластеризация (KMeans).
- Классификация и анализ эмоций.
- Ограничения языковых моделей.

**Рекомендации:** учитывать, что стоп-слова могут быть критичны в некоторых доменах. В лабораторной работе проводить анализ ошибок кластеризации.

**Типичные ошибки:** игнорирование лемматизации; неправильный выбор k в KMeans; непонимание ограничений BERT.

#### 5.4 Модуль 4. Глубокое обучение и нейросетевые подходы

**Ключевые темы:**
- Архитектура Transformer.
- BERT и contextual embeddings.
- Bi-Encoder vs Cross-Encoder.
- Векторный поиск и FAISS.
- Query Understanding.
- Генеративное переформулирование.

**Рекомендации:** визуализировать механизм Self-Attention. В лабораторных работах использовать GPU (Colab Pro, Kaggle). В контрольной работе показывать промежуточные вычисления.

**Типичные ошибки:** путаница в механизмах Bi-Encoder и Cross-Encoder; неправильный расчёт NDCG; непонимание причин меньшей скорости Cross-Encoder.

#### 5.5 Модуль 5. Специализированные виды поиска

**Ключевые темы:**
- Персонализация поиска.
- Профили пользователей.
- Поведенческая сегментация.
- Мультимодальный поиск.
- Корпоративный поиск.
- Этические проблемы.
- Информационный пузырь.
- Холодный старт.

**Рекомендации:** учитывать баланс релевантности и разнообразия. В этическом анализе выбирать актуальные проблемы и предлагать реалистичные решения.

**Типичные ошибки:** смешение персонализации и адаптации; непонимание информационного пузыря; поверхностный анализ этических проблем.

### 7. Правила использования больших языковых моделей (LLM)

В рамках выполнения лабораторных работ, подготовки к контрольным и экзамену допускается использование больших языковых моделей при соблюдении следующих правил.

#### 7.1 Разрешённое использование

| Сценарий | Разрешено | Требования |
| :--- | :---: | :--- |
| Поиск информации и справочных данных | Да | Указывать источник в отчёте |
| Генерация идей и альтернативных подходов | Да | Описывать в отчёте |
| Проверка синтаксиса и отладка кода | Да | Самостоятельно понимать код |
| Объяснение концепций и терминов | Да | — |
| Генерация шаблонов кода | Ограниченно | Код должен быть адаптирован и понят студентом |

#### 7.2 Запрещённое использование

| Сценарий | Запрещено | Последствия |
| :--- | :---: | :--- |
| Полная генерация кода лабораторной работы | Да | Работа не принимается |
| Написание ответов на теоретические вопросы | Да | Оценка снижается |
| Решение задач на расчёт метрик | Да | Работа не принимается |
| Генерация выводов и анализа без самостоятельной работы | Да | Оценка снижается |
| Использование LLM во время экзамена | Да | Нарушение академической этики |

#### 7.3 Требования к цитированию

При использовании LLM в лабораторных работах необходимо:
1. Указывать факт использования в разделе «Методология».
2. Указывать модель и дату обращения.
3. Приводить промпт в приложении к отчёту.

**Пример оформления:**
> При выполнении работы использовалась модель ChatGPT-4 для уточнения синтаксиса библиотеки PyTorch. Промпт: «Как выполнить fine-tuning BERT для задачи классификации в PyTorch?» от 15.10.2024. Генерация полного кода не производилась, решения адаптированы самостоятельно.

#### 7.4 Проверка на использование

Преподаватель оставляет за собой право:
- проводить устное собеседование по коду для проверки понимания;
- запрашивать историю промптов;
- снижать оценку при обнаружении некорректного использования.

#### 7.5 Этические принципы

1. Академическая честность: студент несёт ответственность за все предоставленные материалы.
2. Понимание материала: использование LLM не заменяет понимание.
3. Справедливость: одинаковые правила для всех студентов.


### 8. Часто задаваемые вопросы

**Вопрос:** Как выбрать между pointwise, pairwise и listwise?

**Ответ:** Выбор зависит от данных и задачи:
- Pointwise — при наличии независимых оценок релевантности для каждого документа.
- Pairwise — при наличии парных сравнений.
- Listwise — для прямой оптимизации метрики списка (NDCG).

**Вопрос:** В чём разница между Bi-Encoder и Cross-Encoder?

**Ответ:** Bi-Encoder кодирует запрос и документ отдельно (быстро, подходит для первого этапа). Cross-Encoder кодирует их совместно (медленнее, но точнее, подходит для второго этапа).

**Вопрос:** Как бороться с Position Bias?

**Ответ:** Использовать позиционные признаки, методы коррекции смещения (IPW), данные A/B-тестов для калибровки.

**Вопрос:** Что такое информационный пузырь и как его избежать?

**Ответ:** Информационный пузырь возникает при чрезмерной персонализации. Способы смягчения: механизмы разнообразия в ранжировании, баланс релевантности и новизны.

**Вопрос:** Можно ли использовать ChatGPT при выполнении лабораторных работ?

**Ответ:** Да, с ограничениями. Допускается использование для уточнения синтаксиса, поиска информации, генерации идей. Запрещена полная генерация кода и выводов. Промпты должны быть задокументированы. Подробнее см. раздел 7.
