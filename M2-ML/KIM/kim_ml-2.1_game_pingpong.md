# Индикатор ML-2.1

Различает подходы к обучению ранжированию (pointwise, pairwise, listwise)

**Занятие:** 2.1. Введение в машинное обучение для поиска

**Тип контроля:** текущий

**Форма:** Деловая игра «Архитектурный пинг-понг»

**Максимальный балл:** 10 (в составе блока текущего контроля)

---

## Описание задания

Студенты разбиваются по парам. На листе нарисована таблица: строка — этап работы ML-инженера (Сбор данных → Выбор функции потерь → Оптимизация метрики → A/B-тест), столбец — три подхода (Pointwise, Pairwise, Listwise).

- **Раунд 1 (3 мин):** Первый студент заполняет одну любую ячейку для Pointwise (например: «Функция потерь: MSE»). Передает лист партнеру.
- **Раунд 2 (3 мин):** Второй студент должен заполнить аналогичную ячейку для Pairwise, но указать принципиальное отличие (например: «Функция потерь: попарное сравнение вероятностей RankNet»).
- **Раунд 3 (3 мин):** Первый студент переходит к Listwise.
- **Финал (6 мин):** Пары обмениваются листами с другой парой. Задача — найти в чужой таблице логическую ошибку или дублирование.
