## Карта измерительного материала

| Параметр | Значение |
| :--- | :--- |
| **Код компетенции** | ML-2.2 |
| **Индикатор** | Способен понимать архитектурные особенности моделей семантического поиска (DSSM, CDSSM, Bi-encoder, Cross-encoder) |
| **Форма контроля** | Практическое задание (аудиторная работа с карточками) |
| **Время выполнения** | 15 минут |
| **Максимальный балл** | 10 |
| **Тип заданий** | Идентификация архитектур по описанию оценки |


## Спецификация

| № | Проверяемый элемент | Уровень сложности | Балл |
| :--- | :--- | :--- | :--- |
| 1 | Идентификация Bi-encoder/Siamese архитектуры | Базовый | 3 |
| 2 | Идентификация Cross-encoder архитектуры | Базовый | 3 |
| 3 | Идентификация CDSSM архитектуры | Базовый | 3 |
| 4 | Обоснование выбора (устное пояснение) | Повышенный | 1 |


## Условия выполнения

**Формат работы:** Фронтальная работа. У каждого студента есть карточки «А», «Б», «В».

**Материалы для предъявления:** Преподаватель выводит на экран 3 пары [Запрос — Документ]. Для каждой пары показаны два варианта оценки релевантности от разных моделей.

**Описания вариантов:**

- **Вариант А (Bi-encoder/Siamese):** Косинусное сходство усредненных векторов. Оценка высокая, но текст содержит слова-синонимы, которых нет в запросе.
- **Вариант Б (Cross-encoder):** Проход [Query+Doc] через BERT. Оценка ниже, так как модель увидела отсутствие прямого взаимодействия слов, но уловила отрицание.
- **Вариант В (Ложный CDSSM):** Свертка по буквам. Оценка очень высокая при наличии опечатки в запросе.

**Регламент:**

1. Студенты видят пару и три оценки.
2. За 90 секунд они должны решить, какая архитектура стоит за каждым вариантом, исходя из того, как модель обрабатывает контекст, порядок слов и шум.
3. На счет «три» все поднимают карточку одновременно.


## Критерии оценивания

| Критерий | Макс. балл | Описание |
| :--- | :--- | :--- |
| Правильная идентификация (за каждую пару) | 3 (×3 = 9) | Верно определена архитектура для каждого из трех вариантов |
| Обоснование выбора | 1 | Устное пояснение, демонстрирующее понимание принципов работы архитектур |


## Ответы и ключи

| Пара | Правильный ответ | Обоснование |
| :--- | :--- | :--- |
| **Пара 1** (синонимы) | **А (Bi-encoder)** | Bi-encoder кодирует запрос и документ независимо, поэтому не видит прямого взаимодействия слов. Синонимы дают высокую косинусную близость, даже если точных совпадений нет. |
| **Пара 2** (отрицание) | **Б (Cross-encoder)** | Cross-encoder видит конкатенацию запроса и документа, поэтому может уловить отрицание как паттерн взаимодействия. Оценка ниже из-за отсутствия прямых совпадений, но отрицание распознано. |
| **Пара 3** (опечатка) | **В (CDSSM)** | CDSSM работает на уровне буквенных n-грамм, поэтому опечатки не снижают оценку радикально — модель видит схожесть на уровне подстрок. |

**Типичные ошибки студентов:**

- Путают Bi-encoder и Cross-encoder (не учитывают, что Cross-encoder видит взаимодействие слов).
- Не учитывают, что CDSSM устойчив к опечаткам из-за работы на уровне букв.
- Предполагают, что BERT (Cross-encoder) всегда дает более высокие оценки.
