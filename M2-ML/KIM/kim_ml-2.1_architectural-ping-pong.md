## Карта измерительного материала

| Параметр | Значение |
| :--- | :--- |
| **Код компетенции** | ML-2.1 |
| **Индикатор** | Способен формулировать задачу ранжирования как задачу машинного обучения, различать pointwise, pairwise и listwise подходы |
| **Форма контроля** | Практическое задание (аудиторная работа в парах) |
| **Время выполнения** | 15 минут |
| **Максимальный балл** | 10 |
| **Тип заданий** | Заполнение таблицы, анализ и поиск ошибок |


## Спецификация

| № | Проверяемый элемент | Уровень сложности | Балл |
| :--- | :--- | :--- | :--- |
| 1 | Заполнение ячеек таблицы для всех трех подходов | Базовый | 5 |
| 2 | Корректное определение принципиальных отличий между подходами | Повышенный | 3 |
| 3 | Выявление логических ошибок или дублирования в чужой таблице | Повышенный | 2 (max) |


## Условия выполнения

**Формат работы:** Студенты разбиваются по парам. Каждая пара получает лист с таблицей.

**Таблица для заполнения:**

| | Pointwise | Pairwise | Listwise |
| :--- | :--- | :--- | :--- |
| **Сбор данных** | | | |
| **Выбор функции потерь** | | | |
| **Оптимизация метрики** | | | |
| **A/B-тест** | | | |

**Регламент:**

- **Раунд 1 (3 мин):** Первый студент заполняет одну любую ячейку для Pointwise. Передает лист партнеру.
- **Раунд 2 (3 мин):** Второй студент заполняет аналогичную ячейку для Pairwise, указывая принципиальное отличие.
- **Раунд 3 (3 мин):** Первый студент переходит к Listwise, заполняя соответствующую ячейку.
- **Финал (6 мин):** Пары обмениваются листами с другой парой. Задача — найти в чужой таблице логическую ошибку или дублирование.


## Критерии оценивания

| Критерий | Макс. балл | Описание |
| :--- | :--- | :--- |
| Полнота заполнения таблицы | 5 | Заполнены все ячейки в рамках своей пары (минимум 4 ячейки) |
| Корректность определений | 3 | Верно указаны принципиальные различия между подходами |
| Качество рецензирования | 2 | Найдены логические ошибки или дублирования в чужой таблице (каждая ошибка — 1 балл, но не более 2) |


## Ответы и ключи

**Примеры корректного заполнения:**

| | Pointwise | Pairwise | Listwise |
| :--- | :--- | :--- | :--- |
| **Сбор данных** | Логи с релевантностью каждого документа | Пары документов (релевантный vs нерелевантный) | Списки документов с ранжировкой |
| **Выбор функции потерь** | MSE, MAE, LogLoss | Cross-entropy на парах, RankNet | LambdaRank, ListNet, ListMLE |
| **Оптимизация метрики** | Минимизация ошибки предсказания | Минимизация количества инверсий | Прямая оптимизация NDCG |
| **A/B-тест** | Сравнение средних оценок | Сравнение доли релевантных в топе | Сравнение NDCG по всей выдаче |

**Типичные ошибки для поиска:**

- Использование метрики NDCG в pointwise-подходе (NDCG — метрика для списка, а не для отдельного документа).
- Отсутствие группировки по query_id в pointwise (pointwise рассматривает документы независимо).
- Смешение понятий: использование pairwise функции потерь в listwise подходе.
- Утверждение, что pairwise оптимизирует NDCG напрямую (это не так).


## Эталон выполнения

- **Сбор данных (Pointwise):** Каждый документ размечен независимо. Пример: query — «кофе», документ — «зерна арабика», оценка — 4 из 5.
- **Сбор данных (Pairwise):** Формируются пары. Пример: для запроса «кофе» документ А (про зерна) > документ Б (про чай).
- **Сбор данных (Listwise):** Полный список документов для запроса с известным идеальным порядком. Пример: [doc1, doc2, doc3] с правильным ранжированием.
