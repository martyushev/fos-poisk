# Сквозной командный проект
В ходе изучения дисциплины студент выполняет лабораторные занятия, задания которых соответствуют последовательным этапам командного проекта **«Разработка интеллектуальной поисковой системы для предметной области»**.

## 1. Общее описание проекта

**Тип проекта:** командный (3–4 человека), сквозной, выполняемый в течение семестра параллельно с лабораторными работами в рамках самостоятельной работы студента.

**Цель проекта:**  
Разработать прототип поисковой системы, которая принимает текстовый запрос пользователя и возвращает ранжированный список релевантных документов из заданного корпуса. В ходе разработки системы должны быть реализованы все этапы - от статистических моделей до нейросетевого ранжирования и персонализации.

**Формат защиты проекта:**  
- Наличие рабочей системы.  
- Исходный код и документация в репозитории.  
- Финальная презентация (10–15 мин) с демонстрацией работы системы и сравнительным анализом качества на каждом этапе.  
- Отчёт с метриками (NDCG@10, MAP, Precision@5) для всех реализованных моделей.

## 2. Технологический стек

| Компонент | Технологии |
| :--- | :--- |
| Язык программирования | Python |
| Предобработка текста | NLTK, pymorphy2 (для русского), spaCy |
| Векторизация и классические модели | scikit-learn, Gensim |
| Ранжирование (ML) | LightGBM / XGBoost, RankNet (ручная реализация или catboost) |
| Нейросетевые модели | Hugging Face Transformers (BERT, DistilBERT, Sentence-BERT) |
| Индексация и поиск | FAISS (для векторного поиска), собственный инвертированный индекс |
| Веб-интерфейс (опционально) | FastAPI / Flask + HTML-шаблоны |
| Визуализация и анализ | Jupyter Notebook (для экспериментов), matplotlib / seaborn |
| Сборка и зависимостями | Docker (опционально), requirements.txt, Makefile |

## 4. Этапы выполнения

| Этап проекта | Описание | Результат |
| :--- | :--- | :--- |
| **№1.** Исследование классических моделей | **Спринт 1.** Сбор корпуса, предобработка, реализация инвертированного индекса, TF-IDF, BM25. Формирование бенчмарка (метрики на тестовых запросах). | Базовый поисковый движок + файл с метриками baseline. |
| **№2.** Реализация ML-ранжирования | **Спринт 2.** Извлечение признаков (статистические, текстовые), обучение LightGBM / RankNet, сравнение с BM25. | Обученный ранкер, пайплайн признаков, отчёт с улучшением метрик. |
| **№3.** NLP для запросов и документов | **Спринт 3.** Добавление модулей исправления опечаток, расширения запросов, кластеризации документов. Интеграция новых признаков в ранкер. | Расширенный пайплайн обработки, повторная оценка качества. |
| **№4.** Нейросетевое ранжирование | **Спринт 4.** Fine-tuning BERT / использование Sentence-BERT для эмбеддингов. Реализация семантического поиска через FAISS. | Нейросетевой ранкер, сравнение с классическими моделями. |
| **№5.** Персонализация и адаптация | **Спринт 5.** Сбор поведенческих данных (имитация), построение профилей пользователей, переранжировка с учётом предпочтений. | Прототип персонализированной выдачи, демонстрация для 3–5 профилей. |

## 5. Требования к проекту

### Функциональные требования:
1. Система принимает текстовый запрос и возвращает топ-10 документов.
2. Поддерживается переключение между режимами ранжирования (BM25, ML-ранкер, BERT).
3. Реализована базовая предобработка для русского и/или английского языка.
4. Ведётся логирование запросов и кликов (для персонализации).
5. Есть возможность сравнения метрик для разных моделей на одном тестовом наборе.

### Нефункциональные требования:
1. Время обработки запроса — не более 2 секунд (для прототипа).
2. Код документирован (docstrings, комментарии на сложные участки).
3. Проект сопровождается README с инструкцией по установке и запуску.
4. Эксперименты воспроизводимы (фиксированный seed, версии библиотек).

### Критерии оценки:
- **Качество реализации** (40%) — код работает, соответствует архитектуре, обработка ошибок.
- **Результативность** (30%) — достигнуто улучшение метрик на каждом этапе (от BM25 к BERT).
- **Качество анализа** (20%) — в отчёте присутствует содержательное сравнение моделей, визуализации, выводы.
- **Командная работа** (10%) — разделение ролей, использование Git, вклад каждого участника прозрачен.

## 7. Рекомендации по выбору темы

- **Для сильных команд** (хорошо владеют ML и NLP) — выбрать темы с нестандартной структурой данных: **#4 (юридические документы)**, **#16 (финансовая отчётность)**, **#20 (поиск по коду)**.
- **Для команд с инженерным уклоном** — выбрать темы, где важен краулинг и построение индекса: **#3 (новости)**, **#9 (музыка)**, **#10 (отзывы)**.
- **Для команд, интересующихся персонализацией** — выбрать темы с выраженным поведенческим аспектом: **#5 (медицина)**, **#13 (соцсети)**, **#17 (спорт)**.
- **Для команд с аналитическим уклоном** — выбрать темы, где важна интерпретация результатов: **#6 (вакансии)**, **#7 (образование)**, **#15 (техническая документация)**.

## Заключение

Сквозной проект позволяет студентам пройти полный цикл разработки поисковой системы в рамках одной выбранной темы. Командный формат развивает навыки работы в команде, использование Git, декомпозицию задач и интеграцию компонентов. Выбор темы может быть персональным или из 20 предлагаемых вариантов. Это даёт возможность учесть интересы каждой команды и продемонстрировать результаты в выбранной предметной области.
