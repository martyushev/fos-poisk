# Фонд оценочных средств по дисциплине «Модели и методы интеллектуального поиска»

Этот репозиторий предназначен для открытого и воспроизводимого представления фонда оценочных средств (ФОС) по дисциплине «Модели и методы интеллектуального поиска». Здесь объединены рабочая программа дисциплины, модель измерения результатов обучения, контрольно-измерительные материалы (КИМ), критерии оценивания, методические рекомендации и используемые образовательные ресурсы. Результаты обучения связываются с компетентностно-ролевой моделью в области искусственного интеллекта (КРМ версии 3.0).

## 1. О дисциплине

**Цель дисциплины:** сформировать у студентов теоретические знания и практические навыки разработки, настройки и оценки интеллектуальных поисковых систем, использующих методы машинного обучения, обработки естественного языка, векторного поиска, больших языковых моделей и RAG подходов для решения прикладных задач информационного поиска и поддержки принятия решений.

**Основные задачи:** 
- освоить принципы построения современных интеллектуальных поисковых систем и технологий информационного поиска, изучить методы и алгоритмы обработки естественного языка, применяемые в поиске;
- освоить современные подходы NLP, эмбеддингов и семантического поиска;
- научиться проектировать и реализовывать RAG-сценарии на основе текстового корпуса;
- развить навыки применения методов классификации, кластеризации и анализа текста;
- получить опыт работы с инструментами и библиотеками для разработки интеллектуальных поисковых систем;
- изучить методы анализа и интерпретации результатов работы моделей;
- сформировать способность оценивать качество поисковой выдачи с помощью метрик;
- сформировать понимание этических аспектов использования персональных данных и конфиденциальности в поисковых системах.

**Структура и содержание.** Дисциплина состоит из 5 модулей.
- **Модуль 1.** Введение в интеллектуальный поиск.
- **Модуль 2.** Машинное обучение в поиске.
- **Модуль 3.** Обработка естественного языка для поисковых систем.
- **Модуль 4.** Глубокое обучение и нейросетевые подходы.
- **Модуль 5.** Специализированные виды поиска.

Обучение включает лекции, лабораторные работы и работу над проектом. Итоговая оценка формируется по совокупности следующих компонентов:
- промежуточные тестовые задания по темам дисциплины;
- защита лабораторных работ, являющихся этапами сквозного проекта по разработке поисковой системы;
- защита проекта с демонстрацией работоспособности системы, документации, репозитория;
- экзамен.

**Пререквизиты дисциплины:**
- «Технологии создания программных продуктов» (LC-3),
- «Машинное обучение» (ML-2, ML-3),
- «Рекомендательные системы» (LC-3, ML-2, ML-3).

**Кореквизиты дисциплины:**
- «Технологии обработки естественного языка» (ML-2).

**Постреквизиты дисциплины:**
- «Системы поддержки принятия решений» (LC-3, ML-3).

**Планируемые результаты обучения.**

- **Проектировать и реализовывать RAG-сценарии.** По завершении дисциплины обучающийся сможет проектировать конвейеры Retrieval-Augmented Generation (RAG) на основе заданного текстового корпуса: настраивать этапы индексации, извлечения релевантных фрагментов с помощью векторного поиска и генерации ответов с использованием больших языковых моделей для прикладных задач.
- **Применять методы обработки естественного языка к данным.** Обучающийся сможет применять современные подходы NLP (токенизация, лемматизация, стемминг), генерировать семантические эмбеддинги текстов с помощью предобученных трансформеров и использовать их для построения систем семантического поиска, а также решать задачи тематического моделирования, классификации и кластеризации документов.
- **Разрабатывать прототипы интеллектуальных поисковых систем.** Обучающийся сможет разрабатывать сквозные прототипы поисковых сервисов, используя профильный стек инструментов (например, Hugging Face Transformers, Sentence-Transformers, FAISS или Elasticsearch vector search, LangChain/LlamaIndex), включая пайплайны препроцессинга данных, индексирования и ранжирования.
- **Оценивать качество поисковой выдачи и интерпретировать результаты.** Обучающийся сможет рассчитывать и анализировать метрики качества информационного поиска (Precision@k, Recall@k, F1-score, nDCG, MRR, MAP), проводить A/B-тестирование алгоритмов ранжирования, выполнять анализ ошибок, логировать запросы пользователей и интерпретировать решения моделей методами объяснимого ИИ (LIME, SHAP).
- **Выбирать и настраивать архитектуры ранжирования под задачу.** Обучающийся сможет обосновывать выбор между классическими моделями BM25, нейросетевыми кросс-энкодерами (Cross-Encoders), би-кодерами (Bi-Encoders) и гибридными схемами, а также выполнять их тонкую настройку (fine-tuning) под специфику предметной области.
- **Обеспечивать этичное использование данных в поисковых системах.** Обучающийся сможет выявлять риски утечки персональных данных при построении индексов, оценивать предвзятость (bias) моделей по отношению к социальным группам и темам, внедрять базовые механизмы анонимизации, фильтрации чувствительного контента и соблюдать принципы конфиденциальности при разработке информационных систем.

Полное описание приведено в [РПД](docs/rpd.md).
